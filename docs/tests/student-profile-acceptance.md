# 学生档案模块验收测试矩阵

> 适用范围：学生档案列表、详情抽屉、全屏模式、历史事项聚合、学籍动态和跨模块进入。
> 当前优先实现 Phase 5.2A：共享数据基线 + 核心只读详情。后续阶段继续复用本矩阵。

## 1. 模块边界

| 编号 | 验收项 | 预期结果 | 阶段 |
|---|---|---|---|
| SP-001 | 学生档案定位 | 模块只负责查询和聚合，不直接新增、编辑或删除来源业务记录 | 全局 |
| SP-002 | 干预记录导航 | 不再存在独立“干预记录”一级导航；干预数据保留在预警详情、学生档案和归档中 | Phase 5.1 |
| SP-003 | 组织人员管理 | 不出现学校、年级、班级、学生或教师账号 CRUD | 全局 |
| SP-004 | 专业判断边界 | 不生成下一步专业处置建议，不替心理老师判断 | 全局 |
| SP-005 | 当前权限 | 当前 MVP 中所有心理老师均可查看完整学生档案 | 全局 |

## 2. 学生档案列表

| 编号 | 输入 / 操作 | 预期结果 | 禁止结果 |
|---|---|---|---|
| SP-L001 | 进入学生档案 | 默认只显示在校学生 | 默认混入毕业、离校学生 |
| SP-L002 | 查看默认表格 | 只显示学生信息、年级/班级、操作 | 显示风险、状态、负责人、测评或干预详情 |
| SP-L003 | 姓名搜索 | 支持模糊匹配 | 必须输入完整姓名 |
| SP-L004 | 学号搜索 | 支持精确或前缀匹配 | 仅按姓名搜索 |
| SP-L005 | 同名学生 | 使用学号、年级、班级清晰区分 | 仅显示姓名导致误选 |
| SP-L006 | 选择年级 | 班级选项只显示对应年级 | 显示全部班级 |
| SP-L007 | 清除年级 | 自动清除已失效班级条件 | 保留无效班级 |
| SP-L008 | 高级筛选 | 支持当前风险、当前状态、是否有活动预警、是否有历史干预、负责心理老师、在校状态 | 只能筛选表格中可见列 |
| SP-L009 | 多类筛选 | 不同类别之间为 AND | 使用 OR 导致结果过宽 |
| SP-L010 | 同类多选 | 同一类别内部为 OR | 多选后无结果 |
| SP-L011 | 清空高级筛选 | 不清空搜索关键词 | 同时清空全部输入 |
| SP-L012 | 全部重置 | 清空搜索和全部筛选 | 仍保留隐藏条件 |
| SP-L013 | 无搜索结果 | 展示搜索无结果和调整提示 | 空白页面 |
| SP-L014 | 无筛选结果 | 展示已选条件和清空入口 | 无法恢复 |
| SP-L015 | 点击查看档案 | 打开正确学生的详情抽屉 | 打开同名其他学生 |

## 3. 详情抽屉

| 编号 | 学生情况 | 必须展示 | 禁止操作 |
|---|---|---|---|
| SP-D001 | 有活动事项 | 学生信息、当前概况、当前活动事项、历史记录入口 | 直接改变预警状态 |
| SP-D002 | 无活动事项 | 明确显示“暂无活动事项” | 用最近闭环状态冒充当前状态 |
| SP-D003 | 有多次历史事项 | 按 caseId 分组展示 | 把不同事项混成一条流程 |
| SP-D004 | 无历史记录 | 保留基础信息并展示各模块空状态 | 隐藏整个档案 |
| SP-D005 | 多次干预 | 全部保留，最新在前 | 最新记录覆盖历史 |
| SP-D006 | 多次复测 | 全部保留，并能看出与对应干预/转介关系 | 复测成为孤立记录 |
| SP-D007 | 已转班/升年级 | 当前班级与历史发生时班级分开显示 | 用当前班级覆盖历史 |
| SP-D008 | 已毕业/离校 | 历史档案仍可通过在校状态筛选查看 | 删除历史档案 |
| SP-D009 | 查看来源记录 | 可以进入对应预警或其他来源记录 | 在档案页复制完整编辑表单 |
| SP-D010 | 关闭抽屉 | 列表搜索、筛选、排序和分页保持 | 返回后条件丢失 |

## 4. 全屏模式

| 编号 | 操作 | 预期结果 |
|---|---|---|
| SP-F001 | 抽屉点击全屏 | 使用同一学生、同一数据切换为全屏布局 |
| SP-F002 | 全屏布局 | 允许不同于抽屉的两列或主从排版 |
| SP-F003 | 切回抽屉 | 保留当前学生、已展开模块和记录位置 |
| SP-F004 | 全屏关闭 | 返回原来源上下文 |
| SP-F005 | 数据更新 | 抽屉和全屏不维护两份独立数据 |
| SP-F006 | 阅读行为 | 全屏切换、展开和查看不写入业务时间线 |

## 5. 当前事项与历史事项

| 编号 | 规则 | 预期结果 |
|---|---|---|
| SP-C001 | 活动事项 | 只使用未闭环、未结束处理的事项 |
| SP-C002 | 当前风险 | 只来自当前活动事项 |
| SP-C003 | 当前状态 | 只来自当前活动事项 |
| SP-C004 | 历史闭环 | 只进入历史记录，不作为当前状态 |
| SP-C005 | caseId | 测评、预警、反馈、干预、复测、转介和闭环记录关联同一事项 |
| SP-C006 | 不同 caseId | 不混合为同一连续处置流程 |
| SP-C007 | 事项顺序 | 单个事项可看出“线索/测评→预警→干预→复测→继续处理/转介/闭环” |

## 6. 学籍动态

| 编号 | 事件 | 预期结果 |
|---|---|---|
| SP-E001 | 转班 | 关闭旧学籍阶段并新增新阶段 |
| SP-E002 | 升年级 | 保留旧年级/班级和起止时间 |
| SP-E003 | 当前阶段 | 结束时间显示“至今” |
| SP-E004 | 毕业/离校 | 默认列表隐藏，但可通过筛选查看 |
| SP-E005 | 历史业务记录 | 继续保留发生时年级和班级 |

## 7. 跨模块进入与返回

| 编号 | 来源 | 进入要求 | 返回要求 |
|---|---|---|---|
| SP-X001 | 左侧导航 | 打开学生档案列表 | 无 |
| SP-X002 | 工作台 | 带入 studentId，直接打开正确档案 | 返回工作台原上下文 |
| SP-X003 | 预警管理 | 带入 studentId、warningId、returnContext，并定位当前事项 | 恢复预警筛选、分页和抽屉 |
| SP-X004 | 查看归档记录 | 学生档案完成后可定位对应 warning/case 归档 | 本阶段只保留接口 |
| SP-X005 | 查看完整复测记录 | 学生档案完成后可定位对应复测记录 | 本阶段只保留接口 |
| SP-X006 | 独立干预记录页 | 不再作为入口或返回目标 | 不新增该页面 |

## 8. 数据一致性

| 编号 | 检查项 | 预期结果 |
|---|---|---|
| SP-I001 | 学生档案数据 | 从预警、测评、反馈、干预、复测和转介来源数据聚合 |
| SP-I002 | 重复副本 | 不建立可独立编辑的第二份业务记录 |
| SP-I003 | 排序 | 历史记录统一最新在前 |
| SP-I004 | 必要字段 | 每条记录保留 studentId、caseId、recordId、occurredAt、recordedBy、sourceModule |
| SP-I005 | 局部失败 | 单个记录模块失败不阻塞整个档案 |
| SP-I006 | 无活动事项 | 列表高级筛选和详情概况理解一致 |
