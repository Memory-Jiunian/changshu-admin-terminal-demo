# 工作台页面 PRD

## 页面结构

- 标题、当前日期、当前心理老师
- 当前待办、逾期事项、今日提醒概览
- 六类待办类型筛选
- 今日复测提醒
- 当前待办列表
- 排序说明、空状态和数据异常

每条待办显示学生、班级、风险等级、任务类型、事实原因、触发或截止时间、负责人和“去处理”。提醒使用“查看安排”，不计入待办。

默认排序为逾期、今天到期、危险到低风险、触发时间。转介任务无截止时间，不标记逾期。

校级总览和系统设置未实现时展示明确占位页，不得回退到预警管理。

反馈模块存在未读记录时显示“标记为已查看”。模块渲染和定位只完成导航，不更新阅读状态。工作台新反馈入口关闭详情时，如果仍有未读记录，使用阻断式确认 Dialog 保护；普通预警入口不触发该保护。

## Phase I1.2 自适应布局

- 工作台不使用居中 `max-width`，占满侧栏之后的主内容宽度。
- `>=1180px` 时，概览下方左侧任务约占 70%，右侧安排约占 30% 且不低于
  320px；两个面板等高并随窗口高度伸缩。
- 桌面页面本身不滚动。任务列表和安排列表各有一个隐藏滚动条的可访问
  滚动区；Tab 横向滚动同样隐藏轨道且不换行。
- `<1180px` 时恢复上下布局和单一页面级滚动，不保留列内滚动。
- 待安排干预文案统一为“事项已形成正式预警，尚未安排干预。”，不改变
  当前老师过滤、任务准入、统计或排序。
