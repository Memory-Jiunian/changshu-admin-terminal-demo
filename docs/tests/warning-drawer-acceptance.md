# 学生风险详情抽屉验收基线

## 1. 使用说明

本文档用于验收预警管理详情抽屉。所有时间线断言遵循两个原则：只展示已发生事件；来源路径不同，前置事件不同。测试数据必须明确其来源路径和已发生事件，不能给每个学生强行补全完整流程。

每个状态都必须保留学生概况、风险依据、处置时间线和底部操作区。风险依据线索类型只能是“班主任上报 / AI倾诉 / 深度测评”。

## 2. 待复核

- [ ] 当前状态显示“待复核”。
- [ ] 展示学生概况、风险依据、处置时间线和底部操作区。
- [ ] 不展示班主任反馈模块。
- [ ] 底部按钮为“驳回 / 继续观察 / 确认正式预警”。
- [ ] 普筛来源至少出现“普筛异常触发”和“普筛异常进入线索池”，且带时间。
- [ ] 班主任上报来源至少出现“班主任上报异常”和“异常线索进入待复核线索池”。
- [ ] 尚未发生补充评估时，不强行展示“发起补充评估”及后续节点。
- [ ] 禁止出现确认正式预警、系统同步协作任务、干预、复测、转介或闭环事件。
- [ ] 反馈状态通常为“未请求”，且不以反馈状态替代当前状态。
- [ ] 线索类型不显示“普筛异常”。

## 3. 观察中

- [ ] 当前状态显示“观察中”。
- [ ] 展示学生概况、风险依据、处置时间线和底部操作区。
- [ ] 不展示班主任反馈模块。
- [ ] 底部按钮为“继续观察 / 确认正式预警 / 驳回”。
- [ ] 时间线出现“发起补充评估 → 深度测评 + AI倾诉完成 → 心理老师复核 → 标记继续观察”。
- [ ] 来源为普筛时，前置事件包含“普筛异常触发”和入池事件；来源为班主任上报时，前置事件包含班主任上报和入池事件。
- [ ] 禁止出现确认正式预警、系统同步协作任务、干预、复测、转介或闭环事件。
- [ ] 不因存在历史事实观察而展示班主任反馈模块。
- [ ] 反馈状态为“未请求”，除非后续规格明确观察阶段可请求协作。
- [ ] 线索类型只允许“班主任上报 / AI倾诉 / 深度测评”。

## 4. 正式预警

- [ ] 当前状态显示“正式预警”。
- [ ] 展示学生概况、风险依据、班主任反馈、处置时间线和底部操作区。
- [ ] 底部按钮为“请求补充反馈 / 记录干预”。
- [ ] 时间线出现“心理老师复核”和“确认正式预警”。
- [ ] 确认正式预警后能证明系统同步了班主任协作任务；独立节点或合并描述的最终形式待确认。
- [ ] 只有心理老师额外请求事实观察后，才出现“请求补充反馈”节点。
- [ ] 禁止出现未实际发生的干预、复测、转介或闭环事件。
- [ ] 没有反馈记录时显示“待反馈”或“反馈超时”，且不显示“查看完整反馈”。
- [ ] 有已读反馈时显示“已反馈”；有未读反馈时显示“有新反馈”。
- [ ] 线索类型只允许“班主任上报 / AI倾诉 / 深度测评”。

## 5. 干预中

- [ ] 当前状态显示“干预中”。
- [ ] 展示学生概况、风险依据、班主任反馈、处置时间线和底部操作区。
- [ ] 底部按钮为“新增干预记录 / 安排复测 / 转介”。
- [ ] 时间线至少包含形成正式预警的已发生节点和“新增干预记录”。
- [ ] 多次干预只追加记录，不覆盖历史节点。
- [ ] 未安排复测时，禁止出现复测提醒、复测完成、查看结果或更新状态。
- [ ] 未发起转介时，禁止出现记录转介结果。
- [ ] 多条班主任反馈按提交时间倒序展示。
- [ ] `hasUnreadFeedback=true` 时显示“有新反馈”；否则显示“已反馈”。
- [ ] 线索类型只允许“班主任上报 / AI倾诉 / 深度测评”。

## 6. 待复测

- [ ] 当前状态显示“待复测”。
- [ ] 展示学生概况、风险依据、班主任反馈、处置时间线和底部操作区。
- [ ] 底部按钮为“查看复测结果 / 更新状态”。
- [ ] “更新状态”只提供“风险解除并闭环 / 继续干预 / 转介”。
- [ ] 时间线出现“安排复测”和“小程序生成班主任待复测提醒”。
- [ ] 到达复测前一天且提醒已发送时，出现“复测前一天提醒班主任”；未到时间时不预生成。
- [ ] 学生实际完成后才出现“学生完成复测”。
- [ ] 心理老师实际查看后才出现“查看复测结果”。
- [ ] 心理老师实际提交判断后才出现“更新复测状态”。
- [ ] 禁止出现“继续预警”状态文案。
- [ ] 反馈模块按反馈记录、未读状态和超时规则展示。
- [ ] 线索类型只允许“班主任上报 / AI倾诉 / 深度测评”。

## 7. 转介中

- [ ] 当前状态显示“转介中”。
- [ ] 展示学生概况、风险依据、班主任反馈、处置时间线和底部操作区。
- [ ] 底部按钮为“记录转介结果 / 安排复测”。
- [ ] 时间线至少出现“发起转介”。
- [ ] 只有实际收到并记录外部反馈后，才出现“记录转介结果”。
- [ ] 记录转介结果不会自动出现“学生完成复测 / 查看复测结果 / 更新复测状态”。
- [ ] 只有点击“安排复测”后才进入待复测，并写入安排复测和提醒节点。
- [ ] 反馈超时时显示“提醒班主任反馈”；没有反馈记录时不显示“查看完整反馈”。
- [ ] 反馈模块按反馈记录、未读状态和超时规则展示。
- [ ] 线索类型只允许“班主任上报 / AI倾诉 / 深度测评”。

## 8. 已闭环

- [ ] 当前状态显示“已闭环”。
- [ ] 展示学生概况、风险依据、班主任反馈、处置时间线和底部操作区。
- [ ] 底部按钮只显示“查看归档记录”。
- [ ] 时间线出现形成闭环所依据的已发生事件、“更新复测状态”和“完成闭环归档”。
- [ ] 闭环事件顺序正确，归档不得早于心理老师的状态判断。
- [ ] 禁止显示新增干预、安排复测、发起转介等会推动当前事项继续流转的按钮。
- [ ] 查看归档记录不新增业务时间线，不修改主处置状态。
- [ ] 历史反馈按记录和已读状态准确展示。
- [ ] 线索类型只允许“班主任上报 / AI倾诉 / 深度测评”。

## 9. 全局反馈用例

- [ ] 有一条或多条反馈记录不自动等于“有新反馈”。
- [ ] 只有 `hasUnreadFeedback=true` 时显示“有新反馈”。
- [ ] 有反馈且 `hasUnreadFeedback=false` 时显示“已反馈”。
- [ ] 多条反馈按 `submittedAt` 时间倒序展示，每条包含提交人、时间和内容摘要。
- [ ] 无反馈时显示“暂无反馈”或“已请求，暂未收到反馈”，不显示“查看完整反馈”。
- [ ] 反馈超时且无反馈时显示“反馈超时”和“提醒班主任反馈”。
- [ ] 反馈状态变化不修改或替代主处置状态。

## 10. 全局流程与隐私用例

- [ ] 转介结果不能自动生成复测结果或复测时间线。
- [ ] 只有执行“安排复测”后才能进入待复测。
- [ ] 普筛异常不能作为风险依据线索类型。
- [ ] 普筛异常只在普筛来源的触发和时间线中出现。
- [ ] 班主任上报路径不出现伪造的普筛异常触发事件。
- [ ] “深度测评 + AI倾诉完成”之前存在“发起补充评估”。
- [ ] 时间线只展示当前学生已经发生的事件。
- [ ] 默认列表仍只有学生信息、风险等级、当前状态、最新动态、发生时间、反馈状态和操作。
- [ ] 默认列表不展示线索类型、下一步动作、负责心理老师、完整测评、完整 AI 对话或完整班主任反馈。
- [ ] 抽屉关闭后，列表筛选状态保持不变。

## 11. Phase 4 前置闸门

以下问题确认前，不应实现对应的本地 mock 状态变化：

- 驳回后的持久化主状态。
- 记录转介结果后的主状态。
- 系统同步班主任协作任务的时间线记录形式。
- 请求补充反馈是否允许选择年级主任。
- AI倾诉是否允许作为独立的直接入池路径。
- 转介、复测更新和闭环后的通知对象。
