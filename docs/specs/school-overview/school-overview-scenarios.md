# 场景化走查卡：管理终端—校级总览（MVP·修订版）

> 来源：功能切片卡《管理终端—校级总览》  
> 修订依据：当前已经冻结的预警、观察反馈、干预预约、复测、转介、工作台和学生档案流程。  
> 本文档用于在页面级 PRD 编写前，检查统计口径、角色权限、隐私边界、数据来源、异常状态和页面职责。  
> 旧版《02A_校级总览_场景化走查卡_MVP》不再作为实现依据。

---

## 0. 本轮修订结论

### 0.1 校级总览的定位

校级总览不是个案处理入口，而是学校层面的态势与资源判断页面：

> 帮助校长和心理老师了解当前学校整体心理风险态势、处置压力和资源需求。

它需要回答：

- 当前测评覆盖是否充分；
- 当前有多少学生被心理老师确认存在中、高、危险风险；
- 风险主要集中在哪些年级或班级；
- 哪些处置阶段存在积压；
- 危险风险、转介、反馈超时和复测未完成的压力有多大；
- 是否需要加强心理老师、班主任协作或外部转介资源。

它不负责回答：

- 某名学生应该如何处置；
- 某名学生是谁；
- 某个个案的测评、倾诉、干预或转介详情是什么；
- 哪名心理老师“工作不到位”。

### 0.2 页面只展示聚合事实

MVP 只展示学校、年级和满足隐私阈值的班级聚合数据。

不展示：

- 学生姓名；
- 学号；
- 联系方式；
- 个案详情入口；
- 完整测评、AI 倾诉、反馈、干预和转介记录。

### 0.3 主状态沿用最新七状态

校级总览统一使用：

```text
待复核
观察中
正式预警
待干预
待复测
转介中
已闭环
```

说明：

- 内部枚举仍可使用 `in_intervention`；
- 页面显示名称必须是“待干预”；
- 预约、改约、取消、未到场和待确认属于子状态，不新增校级主状态。

### 0.4 当前重点关注应纳入，但只展示数量

建议聚合展示：

- 危险风险学生数；
- 正式预警待安排干预事项数；
- 干预预约时间已过、待确认事项数；
- 班主任反馈超时事项数；
- 复测未完成事项数；
- 复测结果待更新事项数；
- 转介中事项数。

这些指标只代表当前事实，不等同于心理老师绩效，也不直接生成资源配置结论。

### 0.5 “没有干预的年级 / 班级”不作为指标

不直接展示：

```text
某年级 / 班级没有干预
```

原因：

- 不是所有风险事项都需要立即干预；
- 观察中、待复核和正式预警处于不同业务阶段；
- 没有干预不能直接推导为工作遗漏；
- 容易被误解为考核指标。

应改为展示：

```text
处置进程分布
+
当前待安排与待确认压力
```

### 0.6 “风险原因”改为“风险线索来源分布”

当前系统已有稳定结构化字段：

- 普筛异常；
- AI 倾诉触发；
- 班主任上报。

因此 MVP 默认展示：

```text
风险线索来源分布
```

暂不从自由文本摘要、AI 会话或班主任反馈中自动推断“风险原因”。

只有后续建立结构化关注主题后，才升级为：

```text
主要关注方向 Top 5
```

## 1. 模块目标

### 1.1 一句话目标

帮助校长和心理老师快速理解：

> 当前学校心理风险的规模、分布、变化和处置压力。

### 1.2 页面职责

校级总览负责：

- 展示当前学期测评覆盖；
- 展示当前心理老师确认风险学生规模；
- 展示风险等级结构；
- 展示年级和班级风险分布；
- 展示处置进程；
- 展示当前重点关注事项；
- 展示当前学期按月变化趋势；
- 展示风险线索来源分布；
- 明确统计周期、更新时间、人数与事项数口径；
- 为校长的资源决策提供事实依据。

### 1.3 页面不负责

- 执行预警、观察、干预、复测、转介或闭环操作；
- 查看具体学生；
- 查看心理老师个人待办；
- 直接评价心理老师工作量或绩效；
- 自动推荐医院或心理老师数量；
- 预测未来风险；
- 替代心理老师进行专业判断。

## 2. 使用者与权限

### 2.1 校长

校长需要知道：

- 全校当前风险规模；
- 哪些年级风险占比更高；
- 危险风险和转介压力；
- 是否存在待安排干预、反馈超时或复测未完成积压；
- 是否需要增加校内外支持资源。

校长权限：

- 查看聚合、脱敏统计；
- 查看学校和年级数据；
- 查看满足隐私阈值的班级数据；
- 不进入任何具体学生详情。

### 2.2 心理老师

心理老师需要知道：

- 全校风险结构；
- 哪些年级或班级需要增加预防工作；
- 当前各处置阶段压力；
- 转介、反馈、干预预约和复测压力；
- 测评覆盖变化。

MVP 中心理老师与校长使用同一聚合页面，不开放个案下钻。

### 2.3 与工作台的差异

工作台：

```text
仅展示当前登录心理老师负责事项
```

校级总览：

```text
在权限范围内聚合全校全部心理老师负责事项
```

不得把工作台“只看我的”过滤规则错误带入校级总览。

## 3. 默认统计范围

### 3.1 时间范围

MVP 默认：

```text
当前学期
```

页面显示：

- 当前统计周期；
- 数据更新时间；
- 当前在校学生数。

MVP 暂不提供任意自定义日期。

趋势按当前学期内自然月展示。

### 3.2 组织范围

默认：

```text
全校
```

支持筛选：

```text
全校
→ 年级
→ 班级
```

切换年级后：

- 清空不再适用的班级选择；
- 所有卡片和图表同步使用同一组织范围；
- 不允许部分模块仍显示全校数据。

### 3.3 人数和事项数必须分开

学生人数用于：

- 在校学生；
- 已完成测评学生；
- 当前确认风险学生；
- 风险等级分布。

事项数用于：

- 七种主状态分布；
- 待安排干预；
- 干预预约待确认；
- 反馈超时；
- 复测未完成；
- 复测结果待更新；
- 转介中；
- 已闭环事项。

页面必须明确显示单位：

```text
人
项
%
```

## 4. 数据来源与唯一真值

### 4.1 上游数据

校级总览依赖：

- 学生在校与学籍数据；
- 测评完成记录；
- 共享 `WarningItem`；
- 心理老师确认风险等级；
- 七种主状态；
- 班主任反馈请求与反馈记录；
- 干预预约与干预结果；
- 复测计划与复测结果；
- 转介与转介跟进；
- 闭环记录；
- 统一时间服务。

### 4.2 不建立第二份统计数据

所有指标通过 selector 从共享业务数据派生。

禁止：

- 在页面组件中手工维护人数；
- 给图表写一份独立 mock 统计数字；
- 用工作台任务数量直接替代校级统计；
- 在不同图表中使用不同时间或组织范围。

### 4.3 当前快照与历史趋势分开

当前快照：

- 使用当前活动事项；
- 已闭环事项不计入当前风险；
- 使用当前最新状态。

历史趋势：

- 使用真实事件时间；
- 保留历史正式预警、闭环和转介事件；
- 不因当前状态变化覆盖历史月份。

## 5. 核心统计口径

### 5.1 测评覆盖率

```text
当前学期完成至少一次有效测评的去重在校学生数
÷
当前在校学生总数
```

展示：

- 在校学生数；
- 已完成测评学生数；
- 尚未完成学生数；
- 测评覆盖率。

同一学生多次测评只计算一人。

### 5.2 当前确认风险学生

统计：

```text
存在当前活动事项
且 confirmedRiskLevel 存在
且等级为中风险 / 高风险 / 危险
的去重学生
```

不统计：

- 只有系统建议风险、尚未完成心理老师复核的事项；
- 已闭环历史事项；
- 已结束且未形成正式预警的线索；
- 学生档案中的低风险谨慎标记。

### 5.3 风险等级分布

仅展示：

```text
中风险
高风险
危险
```

说明：

- 活动预警不存在低风险；
- 低风险只用于测评事实或退出活动预警后的谨慎标记；
- 不得把低风险混入当前活动预警分布。

### 5.4 处置进程分布

当前活动事项：

- 待复核；
- 观察中；
- 正式预警；
- 待干预；
- 待复测；
- 转介中。

历史指标：

- 当前学期已闭环事项。

“已闭环”不与当前活动事项使用同一个百分比分母。

### 5.5 当前重点关注

#### 危险风险学生

```text
当前活动事项 confirmedRiskLevel === critical
的去重学生数
```

#### 待安排干预

```text
currentStatus === formal_warning
且不存在有效干预预约
```

统计全校事项，不使用工作台当前老师过滤。

#### 干预预约待确认

```text
currentStatus === in_intervention
且最新有效预约为 planned
且当前时间超过 plannedAt + 60 分钟
```

只表示：

```text
预约时间已过，等待心理老师确认完成情况
```

不自动等同于学生未到场。

#### 反馈超时

使用统一有效反馈状态：

```text
getEffectiveFeedbackStatus(...) === feedback_overdue
```

#### 复测未完成

```text
currentStatus === pending_retest
最新计划复测尚未完成
当前时间超过 plannedAt + 120 分钟
```

#### 复测结果待更新

```text
最新复测已经完成
且主状态仍为 pending_retest
```

#### 转介中

```text
currentStatus === referral
```

新增转介跟进不会让事项离开转介中。

### 5.6 数据异常

需要单独识别：

- 同一学生多条活动事项；
- 活动预警出现低风险；
- 正式预警及后续状态缺少确认风险等级；
- 待干预缺少有效预约；
- 待复测缺少复测计划；
- 反馈状态与请求 / 反馈记录不一致；
- 学生年级或班级缺失。

异常不得静默计入正常统计。

## 6. 年级和班级风险分布

### 6.1 年级

每个年级展示：

- 在校学生数；
- 当前确认风险学生数；
- 风险学生占比；
- 中 / 高 / 危险结构。

必须同时展示人数和占比，不能只做风险人数排名。

### 6.2 班级

班级作为年级筛选后的二级分布。

隐私阈值：

```text
风险学生数 < 3
→ 不显示精确数字
→ 显示“少量”
```

页面仍可展示该班级学生总数和经过遮蔽的占比区间，但不得帮助校长反推出具体学生。

### 6.3 排序

默认按：

```text
风险学生占比
→ 危险风险人数
→ 高风险人数
```

不是按纯人数排序。

---

## 7. 风险变化趋势

### 7.1 主趋势

当前学期按月展示：

```text
每月新增确认风险学生数
```

去重规则：

- 同一学生在同一月只计算一次；
- 不因为同一学生的多条事项重复计数。

### 7.2 处置趋势

展示：

- 每月新增正式预警事项数；
- 每月闭环事项数；
- 每月新发起转介事项数。

单位明确为：

```text
项
```

### 7.3 趋势解释边界

趋势上升只说明统计事实，不直接表示：

- 学校心理问题恶化；
- 某年级管理不到位；
- 心理老师处置不及时。

可能受到：

- 测评覆盖增加；
- 上报渠道变化；
- 复核效率变化；
- 学期阶段变化

影响。

页面需要提供口径说明。

## 8. 风险线索来源分布

MVP 展示：

- 普筛异常；
- AI 倾诉触发；
- 班主任上报。

口径：

- 按当前确认风险学生的事项来源统计；
- 同一学生存在多条异常活动事项时，先报告数据异常；
- 不从自由文本推断原因。

后续有结构化关注主题后，再新增：

```text
主要关注方向 Top 5
```

## 9. 场景一：校长快速了解学校态势

### 触发

校长打开校级总览。

### 需要回答

- 当前学期测评覆盖怎么样；
- 当前有多少确认风险学生；
- 中、高、危险风险分别有多少；
- 风险主要集中在哪些年级；
- 当前处置链条中哪一阶段压力更大；
- 是否存在危险风险、待安排干预或转介压力。

### 页面优先级

1. 统计周期与更新时间；
2. 测评覆盖率；
3. 当前确认风险学生；
4. 高风险和危险风险学生；
5. 当前重点关注；
6. 年级风险分布；
7. 处置进程分布；
8. 趋势；
9. 风险线索来源。

### 不应出现

- 学生身份；
- 个案入口；
- 专业处置建议；
- 心理老师排名；
- 没有口径说明的数字。

## 10. 场景二：判断是否需要增加心理支持资源

### 触发

校长发现：

- 危险风险学生增加；
- 转介中事项增加；
- 待安排干预增加；
- 干预预约待确认或复测未完成增加；
- 闭环速度低于新增正式预警速度。

### 页面能支持的判断

- 当前风险规模；
- 处置阶段压力；
- 年级分布；
- 转介规模；
- 新增和闭环趋势。

### 页面不能自动得出

- “必须新增一名心理老师”；
- “心理老师工作效率低”；
- “必须签约某家医院”。

资源配置由学校结合学生总数、心理老师数量、个案复杂度、预算和医院资源决定。

## 11. 场景三：寻找需要加强预防的年级

### 操作

```text
选择年级
→ 查看该年级学生基数
→ 查看风险人数、占比和等级结构
→ 查看班级分布
→ 查看线索来源
```

### 正确结果

- 所有指标同步切换；
- 班级小数量被遮蔽；
- 人数和占比同时存在；
- 不显示具体学生；
- 切换年级后旧班级筛选自动清除。

### 错误情况

- 顶部卡片是全校，图表是年级；
- 只用人数判断风险高低；
- 显示少于 3 人的精确班级风险数；
- 从线索来源直接推导风险原因。

## 12. 场景四：查看处置压力

### 页面展示

当前活动状态：

```text
待复核
观察中
正式预警
待干预
待复测
转介中
```

重点关注：

```text
待安排干预
干预预约待确认
反馈超时
复测未完成
复测结果待更新
转介中
```

### 业务解释

- “正式预警”不等于已经安排干预；
- “待干预”表示已经进入干预阶段；
- “预约待确认”不等于学生未到场；
- “待复测”不等于复测已完成；
- “转介中”可包含多次跟进。

页面需要通过 Tooltip 或口径说明避免误解。

## 13. 场景五：查看危险风险和转介压力

页面聚合展示：

```text
危险风险学生：X 人
转介中事项：Y 项
```

可补充：

- 当前学期新发起转介事项；
- 当前转介跟进次数；
- 与上月相比的变化；本轮可暂不实现。

不展示：

- 学生名单；
- 机构详情；
- 转介原因正文；
- 跟进内容。

## 14. 场景六：没有测评或风险数据

### 当前学期尚未开展测评

显示：

```text
当前统计周期暂无有效测评数据
```

并展示当前在校学生数。

### 有测评但没有确认风险

显示：

```text
当前暂无心理老师确认的活动风险事项
```

不得显示：

```text
全校学生均为低风险
```

### 某个年级没有符合口径的数据

显示：

```text
该年级当前暂无符合统计口径的数据
```

不能用 0 暗示“绝对安全”。

---

## 15. 场景七：数据加载失败或局部异常

### 全部失败

展示：

- 校级总览加载失败；
- 重试；
- 数据更新时间不可用。

### 单个模块失败

- 其他模块继续展示；
- 失败模块局部提示和重试；
- 不让单个图表阻塞整页。

### 数据异常

展示聚合级提示，不暴露学生身份：

- 当前统计存在需要核对的数据；
- 本模块已排除异常记录；
- 可由系统管理员或心理老师后续核查。

不得静默修正为正常数值。

---

## 16. 隐私与小数量保护

### 16.1 校长

只能查看：

- 全校；
- 年级；
- 满足阈值的班级聚合。

### 16.2 心理老师

MVP 与校长保持相同聚合视图，不开放学生下钻。

### 16.3 小数量保护

```text
班级风险学生数 < 3
→ 显示“少量”
→ 不展示精确人数
```

不得通过：

- 风险等级组合；
- 单一线索来源；
- 处置阶段

间接反推出个案。

### 16.4 页面行为

以下行为不写入预警处置时间线：

- 打开总览；
- 选择年级 / 班级；
- 切换图表；
- 查看统计口径。

## 20. 页面 PRD 前已冻结的决定

1. 默认统计周期为当前学期。
2. 默认组织范围为全校。
3. 支持年级和班级筛选。
4. 班级风险人数小于 3 时不显示精确值。
5. 校长和心理老师 MVP 都只看聚合数据。
6. 不开放个案下钻。
7. 主趋势为每月新增确认风险学生数。
8. 同时展示新增正式预警、闭环和转介趋势。
9. 默认展示风险线索来源分布，不展示风险原因。
10. 当前重点关注包含待安排干预、干预预约待确认、反馈超时、复测未完成、复测结果待更新和转介中。
11. 校级总览聚合全校数据，不使用工作台“只看当前心理老师”的过滤。
12. “待干预”作为 `in_intervention` 的唯一页面显示名称。

## 21. 完成标准

1. 校长和心理老师能在短时间内理解当前学校风险态势。
2. 人数、事项数和百分比单位明确。
3. 当前快照与历史趋势不混用。
4. 活动预警只统计中、高、危险风险。
5. 已闭环事项不计入当前风险快照。
6. 年级和班级同时展示人数、占比和学生基数。
7. 小数量班级数据执行隐私遮蔽。
8. 工作台当前老师过滤不会影响校级统计。
9. 待安排干预与待干预能够正确区分。
10. 预约待确认不被误写成学生未到场。
11. 复测未完成与复测结果待更新能够正确区分。
12. 转介中允许存在多次跟进。
13. 图表和核心指标使用相同时间与组织范围。
14. 数据异常不会被静默计入正常结果。
15. 页面不展示具体学生或敏感业务记录。
16. 页面不输出专业处置建议或绩效结论。
17. 页面浏览和筛选不写业务时间线。
18. 总览数据从共享业务数据派生，不建立第二份统计真值。
19. 空状态不会暗示“全校绝对安全”。
20. 单个模块失败不会阻塞整页。

## 22. Phase S1.1 校长走查口径

1. 趋势只比较本学期新增正式预警事项与闭环事项，空月份显示 0，未结束月份标明数据截至日期。
2. 顶部处置卡使用“当前处理中 / 本学期已闭环”，不向校长暴露内部“活动事项”术语。
3. 正式预警已有当前反馈轮次、至少一条有效反馈、全部反馈均有 `psychologistReadAt` 且没有有效 planned 预约时，进入“反馈已阅待安排”。
4. 未收到反馈、存在未读反馈、已有预约或已离开正式预警状态时均不计入；该指标不限制心理老师提前预约干预。
5. 待复测事项只有超过计划时间加 120 分钟且仍未完成时才进入“复测超时未完成”；完成后待更新状态不计入。
6. 年级环图展示当前确认风险学生构成，不使用在校生基数计算风险率；年级超过 5 个时合并为前 4 个加其他。
7. 处置成效按真实确认和闭环时间计算；缺失或倒置时间不进入平均周期并形成聚合数据问题。
8. 当前阻塞事项命中反馈超时、干预预约待确认或复测超时任一条件即计入，并按 warningId 去重。
