# 预警管理操作验收矩阵

## 1. 适用范围

本文档验收 Phase 4 本地 mock 操作、业务记录、时间线、列表同步及详情容器一致性。所有操作由负责心理老师执行，不接真实后端或小程序。

## 2. 待复核与观察中

- [ ] 继续观察要求填写观察说明和下次复核时间，成功后进入观察中并新增“标记继续观察”。
- [ ] 观察中再次继续观察保持观察中，并更新观察说明和下次复核时间。
- [ ] 必填项为空时 Dialog 不关闭，已输入内容保留并显示字段错误。
- [ ] 结束处理要求填写原因，成功后 `isActive=false`、`disposition=ended_without_warning`。
- [ ] 结束处理不新增主状态、不映射为已闭环、不删除历史数据。
- [ ] 结束处理后事项从活动列表、Tab 数量及所有筛选结果移除，抽屉关闭。
- [ ] 确认正式预警继续使用风险等级确认弹窗，调整系统提示等级时理由必填。
- [ ] 确认后进入正式预警、反馈变为待反馈，时间线合并正式等级和班主任协作通知。

## 3. 正式预警与干预中

- [ ] 请求补充反馈要求填写反馈要求和截止时间，对象固定为班主任。
- [ ] 请求反馈后主状态仍为正式预警，反馈状态变为待反馈，历史反馈记录保留。
- [ ] 记录干预和新增干预复用同一表单，五个业务字段均必填。
- [ ] 正式预警首次记录干预后进入干预中并生成第一条干预记录。
- [ ] 干预中新增记录保持干预中，不覆盖历史，记录和时间线最新在前。
- [ ] 安排复测后进入待复测并追加未完成复测记录，不预生成学生完成或更新结果事件。
- [ ] 发起转介后进入转介中并追加未完成转介记录，不自动生成结果或复测。

## 4. 待复测

- [ ] 查看最近复测结果不改变状态、最新动态或时间线。
- [ ] 最近复测未完成时显示“复测尚未完成”，且不能提交状态更新。
- [ ] 已完成复测可更新为风险解除并闭环、继续干预或转介。
- [ ] 闭环后进入已闭环，依次新增“更新复测状态”和“完成闭环归档”，历史记录全部保留。
- [ ] 继续干预后进入干预中并新增“更新复测状态”。
- [ ] 转介后进入转介中，自动追加未完成转介记录，并新增“更新复测状态”和“发起转介”。

## 5. 转介中与已闭环

- [ ] 记录转介结果更新最近一条未完成转介记录，状态仍为转介中。
- [ ] 记录转介结果不生成复测记录、不进入待复测、不自动闭环。
- [ ] 转介中只有执行安排复测后才进入待复测。
- [ ] 查看归档记录为只读，展示学生、确认等级、开始和闭环时间、最终结果、记录数量、转介和完整时间线。
- [ ] 查看归档不改变状态、最新动态或时间线；已闭环不显示新增干预、复测或转介操作。

## 6. 通知与阅读行为

- [ ] 确认正式预警、安排复测、发起转介、更新复测状态、完成闭环的描述体现系统已同步通知对应班主任。
- [ ] 年级主任不作为通知或补充反馈对象。
- [ ] 打开抽屉、切换全屏、查看复测结果、查看归档、查看测评或 AI 记录均不写业务时间线。

## 7. 全局回归

- [ ] 每次成功操作同步当前状态、相关反馈状态、最新动态、发生时间、业务记录和时间线。
- [ ] 状态数量、快捷筛选、高级筛选和搜索结果随活动 warning 状态同步变化。
- [ ] 连续点击提交不会生成重复记录；成功后表单清空。
- [ ] 切换学生时关闭未提交 Dialog，不串用表单数据。
- [ ] 时间线、班主任反馈、干预记录和复测记录均最新在顶部，不预生成未来事件。
- [ ] 全屏与抽屉读取同一 warning；全屏操作后列表和返回后的抽屉同步。
- [ ] 抽屉宽度来自公共配置，实际视觉宽度保持 520px 至 560px。
- [ ] 默认列表仍只有七个字段，高级筛选结构和主逻辑不变。
- [ ] 没有新增线索池或学生档案页面。
- [ ] `npm run build` 通过。

## 8. Phase 4.6 协作与复测操作

- [ ] 确认正式预警除风险等级外，反馈要求和反馈截止时间均为必填。
- [ ] 确认截止时间不晚于当前 mock 时间时不能提交，表单输入保留。
- [ ] 确认成功后生成待反馈任务并记录一次反馈请求历史。
- [ ] 无反馈且截止时间未到时不能重复请求，按钮显示“等待反馈”。
- [ ] 无反馈且截止时间已过时可“重新请求反馈”，提交后生成新的请求历史并恢复待反馈。
- [ ] 有反馈后不显示请求反馈操作；已读与未读分别显示已反馈和有新反馈。
- [ ] 请求 Dialog 只读展示班主任姓名和脱敏手机号，不提供年级主任选择。
- [ ] 安排复测的安排时间由系统记录且只读，至少选择一项集中维护的量表。
- [ ] 计划复测时间必须晚于安排时间；成功后记录和时间线均包含量表名称。
- [ ] 查看复测结果和完整复测记录均不写时间线。
- [ ] 归档 Dialog 只读展示全部反馈请求、反馈、干预、复测、转介和完整时间线。
- [ ] 原 29 项状态服务断言继续通过，并新增反馈时效与复测量表断言。

## 9. Phase 5.2B.1 反馈时间线与转介跟进

- [ ] 反馈 requestId 精确关联请求，一轮允许多条回复；无效关联报告异常但保留内容
- [ ] 每条反馈派生稳定时间线事件并与原时间线按 ID 去重
- [ ] 旧转介结果无损迁移为首条 followUpRecord
- [ ] “新增转介跟进”校验时间、记录人和摘要，支持多次追加和重复提交保护
- [ ] 每次跟进更新动态和时间线但保持 referral，不生成复测或闭环事件
- [ ] 安排复测继续按既有规则进入待复测，其他状态流转不变

## 10. Phase I1.1 操作与时间线验收

- [x] 继续观察和重新请求反馈均创建新轮次，旧请求状态正确且时间线格式一致。
- [x] 反馈请求、反馈提交、预约、干预结果、复测和转介事件引用同一结构化记录并稳定去重。
- [x] 不再生成“新增干预记录”标题，兼容旧标题统一显示为“记录干预结果”。
- [x] 宽限期前不能确认未到场；宽限期后可原子记录未到场并重约。
- [x] 取消预约回到正式预警并写入对应时间线。
- [x] 两次复测分别生成安排和完成事件；更新状态只使用最新已完成复测。
