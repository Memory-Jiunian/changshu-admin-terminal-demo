### Phase 3.4 当时的实现差异

- TypeScript `ClueType` 仍包含 `screening_abnormal`，但风险依据组件会回退展示为深度测评，mock 当前未使用该值。本轮禁止修改代码，因此只建立领域约束。
- 部分历史 mock 以“AI 生成风险线索”为起点，没有普筛或班主任上报入池节点。是否保留第三条直接入池路径待确认。
- 当前按钮均为占位反馈，尚未执行状态、最新动态、反馈状态或时间线的本地更新；该工作属于 Phase 4。

## Phase 4：本地 mock 主流程闭环

### 已确认决策

- “结束本次线索处理”不新增主状态，事项转为非活动并从活动列表移除，历史数据保留。
- 活动列表、状态数量、快捷筛选和高级筛选只统计 `isActive=true` 的事项。
- 主要操作使用类型化 action 和独立 Dialog，不再依赖中文按钮文案驱动状态变化。
- 干预、复测、转介均追加业务记录，不覆盖历史记录，不预生成未来事件。
- 确认正式预警、安排复测、发起转介、更新复测状态和完成闭环只在时间线描述中模拟通知班主任；年级主任不接收。
- 查看复测结果、查看归档、查看测评和 AI 记录、打开抽屉及切换全屏均不写时间线。
- 详情抽屉与全屏模式复用同一内容和数据；抽屉宽度收敛为公共配置，供后续模块复用。

### 本轮发现与处理

| 分类 | 发现 | 处理 |
|---|---|---|
| C/D | 旧实现只有确认正式预警会更新本地状态 | Phase 4 集中实现主路径和主要分支的 reducer 式本地更新 |
| D | 旧模型无法表达非活动线索、观察计划、反馈截止和转介历史 | 增量扩展字段并补齐全部 mock |
| E/F | `DrawerActionBar` 通过中文文案触发占位反馈 | 改为只发出 `WarningActionType`，表单和更新由上层协调 |
| E/F | 抽屉宽度硬编码且全屏按钮为占位 | 新增公共详情配置、共享内容组件和真实全屏 Dialog |

### 仍不实现

- 真实后端、登录、权限和小程序通知。
- 线索池页面、学生档案页面和完整独立业务页面。

### 决策原则

在待确认问题得到产品结论前：

1. 不为缺失状态或通知对象自行造规则。
2. 不实现对应的本地 mock 状态变化。
3. 可继续做不依赖这些结论的展示验收和文档校对。
4. 确认后同步更新领域规格、流程规格、PRD、验收用例和 TASKS，再进入实现。

## 2026-07-10：Phase 3.5 预警流程模型校准

### 范围

先同步领域、流程、验收和页面 PRD，再按最新文档整改数据模型、mock 和详情抽屉。只授权“确认正式预警”的局部本地流转，其他按钮继续占位。

### 本轮发现

| 分类 | 发现 | 影响 | 本轮处理 |
|---|---|---|---|
| A/C | 线索池与待复核被混用 | 刚入池、尚未形成证据的事项被错误放进预警主状态 | 明确线索池是上游容器，待复核必须完成补充评估并形成风险依据 |
| A/D | 来源类型和风险依据被同一个 `clueType` 混用 | 普筛异常可能被当作详情证据，高级筛选语义不稳定 | 拆分为 `sourceType` 和 `evidenceTypes` |
| A/D | 系统建议风险等级和心理老师确认等级被同一个 `riskLevel` 混用 | 系统提示可能被误认为专业结论 | 拆分为 `suggestedRiskLevel` 和 `confirmedRiskLevel`，有效等级以后者优先 |
| C/E | 打开详情被误认为完成复核 | 时间线出现没有业务结果的“心理老师复核” | 只有提交三种复核结果才算完成，删除独立复核节点 |
| C/D | 时间线记录过多阅读动作和系统细节 | “查看复测结果”、独立协作任务等噪声掩盖业务结果 | 合并来源入池和正式预警事件，删除阅读节点，统一倒序 |
| D/E | 详情缺少干预记录和复测记录 | 处置过程只能从时间线推断，证据不足 | 新增结构化干预记录与复测记录模块，均按最新在前展示 |
| B | 年级主任权限边界不清楚 | 年级主任可能被错误允许提交事实观察或接收心理处置通知 | 限定为查看协作进度、督办和留痕，不提交班主任反馈，不接收本轮三类通知 |

### 已确认规则

- 三条上游来源路径为普筛异常、班主任上报和 AI倾诉主动触发。
- 正式预警前的深度测评和 AI倾诉属于补充评估，不是复测。
- 确认正式预警必须由心理老师选择正式风险等级；调整系统提示时必须填写理由。
- 确认正式预警与系统生成班主任协作任务合并为一条时间线记录。
- 请求补充反馈只能发给班主任。
- 记录转介结果后保持转介中。
- 发起转介、更新复测状态和完成闭环后通知对应班主任，年级主任暂不接收。
- 业务时间线、班主任反馈、干预记录和复测记录均最新在前。

### 本轮不实现

- 线索池页面和线索池进度交互。
- 真实后端、权限系统和小程序通知。
- 完整干预表单、完整复测结果页和完整归档页。
- 除确认正式预警外的完整本地状态流转。

## 2026-07-14：Phase 4.6 协作、复测与归档收尾

### 本轮发现与决策

| 分类 | 发现 | 最终处理 |
|---|---|---|
| C/D | 首次正式预警只生成笼统协作任务，缺少可执行要求和截止时间 | 确认正式预警必须同时提交反馈要求和晚于确认时间的截止时间 |
| C/D/E | 反馈状态在列表、筛选和反馈卡片中分别判断，超时和重复请求可能不一致 | 建立共享有效反馈状态和操作可用性规则，所有入口统一使用 |
| B/E | 协作对象只写“对应班主任”，且反馈卡片与底部存在重复提醒入口 | 新增独立班主任姓名和脱敏电话；重发入口只保留在底部操作区 |
| D | 复测记录缺少量表，无法解释结果来源和支持归档追溯 | 使用集中量表目录，安排复测至少选择一项，记录保存量表 ID 和名称 |
| A/E | 复测结果弹窗混入心理老师结论，客观结果与处置判断边界不清 | 结果弹窗只展示客观结果；处置判断继续由更新状态操作承接 |
| D/E | 归档只展示概况、转介和时间线，不能完整追溯 | 扩展为只读完整归档，覆盖依据、请求、反馈、干预、复测、转介和时间线 |

### 延后到学生档案

- 从归档入口跳转学生档案并定位当前 warning/case。
- 跨事项汇总学生长期测评、反馈、干预、复测和转介记录。
- 本轮仅保留回调接口，不实现路由或无响应入口。
