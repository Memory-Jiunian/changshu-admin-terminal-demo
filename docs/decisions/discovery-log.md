### 当前实现与新基线的已知差异

- TypeScript `ClueType` 仍包含 `screening_abnormal`，但风险依据组件会回退展示为深度测评，mock 当前未使用该值。本轮禁止修改代码，因此只建立领域约束。
- 部分历史 mock 以“AI 生成风险线索”为起点，没有普筛或班主任上报入池节点。是否保留第三条直接入池路径待确认。
- 当前按钮均为占位反馈，尚未执行状态、最新动态、反馈状态或时间线的本地更新；该工作属于 Phase 4。

### 决策原则

在待确认问题得到产品结论前：

1. 不为缺失状态或通知对象自行造规则。
2. 不实现对应的本地 mock 状态变化。
3. 可继续做不依赖这些结论的展示验收和文档校对。
4. 确认后同步更新领域规格、流程规格、PRD、验收用例和 TASKS，再进入实现。
