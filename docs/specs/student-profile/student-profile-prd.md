# 页面级 PRD：学生心理档案 MVP（优化版）

## 1. 页面目标
帮助心理老师快速定位目标学生，并通过标准详情抽屉或全屏模式查看该学生的基础信息、当前心理状态及历史测评、预警、干预、复测、转介和闭环记录，为当前专业判断提供历史依据。

## 2. 页面范围

> 当前实现切片为 Phase 5.2A，只交付共享数据基线、学生信息、当前心理状态概况、当前活动事项、学籍动态和历史事项摘要。以下全屏、完整专业记录和跨模块能力是后续阶段的长期规格，本轮不实现。

### 包含
- 学生档案列表
- 姓名 / 学号搜索
- 年级 / 班级筛选
- 高级筛选
- 标准详情抽屉
- 全屏查看模式
- 学生基础信息
- 当前心理状态概况
- 当前活动事项
- 学籍动态
- 测评、预警、干预、复测、转介和闭环记录
- 全量业务时间线
- 跨模块进入与返回

### 不包含
- 组织结构和人员账号 CRUD
- 批量导入导出
- 真实校务系统同步
- 真实权限审批
- 在档案页新增、编辑或删除来源业务记录
- 在档案页直接改变业务状态
- 家长联系方式、家庭住址等非必要信息

## 3. 产品边界
1. 学生档案是查询和聚合模块，不是组织人员管理模块。
2. 学生基础信息默认只读。
3. 业务记录由测评、预警、干预、复测和转介模块产生。
4. 档案页只引用这些记录，不生成第二份可独立编辑的数据。
5. 历史专业记录不能删除或覆盖。
6. 当前事项与历史事项必须区分。
7. 系统只呈现事实，不生成专业处置建议。
8. 所有心理老师均可查看完整学生档案；当前 MVP 不实现负责制权限隔离。

## 4. 页面结构

### 4.1 学生档案列表
- 页面标题和说明
- 搜索区
- 快速筛选
- 高级筛选
- 学生列表
- 结果数量和分页
- 空状态
- 历史学生入口

### 4.2 学生档案详情
默认以右侧抽屉打开。

详情结构：
1. 学生信息
2. 当前心理状态概况
3. 当前活动事项
4. 学籍动态
5. 业务事项记录
6. 测评记录
7. 预警记录
8. 干预记录
9. 复测记录
10. 转介与闭环记录
11. 全量业务时间线

## 5. 抽屉与全屏模式

### 5.1 标准抽屉
- 宽度与预警管理详情抽屉一致
- 统一引用项目级详情抽屉宽度配置
- 默认使用单列卡片布局
- 支持模块折叠
- 顶部展示学生身份和全屏按钮
- 抽屉关闭后保留列表搜索、筛选、排序和分页

### 5.2 统一宽度规则
建议在公共布局或设计配置中维护一个统一值，例如：

```ts
DETAIL_DRAWER_WIDTH
```

或统一 Tailwind class / CSS variable。

预警详情、学生档案详情及后续同级详情抽屉都引用同一配置。

修改该配置时，应一次性更新全部同级抽屉，不在各组件中分别硬编码宽度。

### 5.3 全屏模式
全屏模式复用标准抽屉的数据和业务组件，但允许重新组织布局。

建议布局：
- 顶部：学生身份、当前状态、返回抽屉 / 关闭
- 左栏：学生基础信息、当前状态、当前事项、学籍动态
- 右栏：测评、预警、干预、复测、转介和时间线

全屏切换规则：

```text
列表
→ 标准抽屉
→ 全屏模式
→ 标准抽屉或列表
```

切换过程中保留：
- 当前学生
- 已展开模块
- 当前记录定位
- 来源页面上下文

## 6. 用户与权限

| 角色 | 可查看范围 | 可操作 |
|---|---|---|
| 心理老师 | 全部学生完整档案 | 查看、切换显示模式、跳转来源业务记录 |
| 班主任 / 年级主任 | 不使用该管理终端模块 | 无 |
| 校级管理者 | 不进入个体档案 | 未来只查看脱敏汇总 |

当前 MVP 不展示无权限状态，不提供申请查看入口。

## 7. 页面入口与出口

### 7.1 入口

| 来源 | 触发动作 | 带入参数 |
|---|---|---|
| 左侧导航 | 点击学生档案 | 无 |
| 工作台 | 点击学生 | studentId |
| 预警管理 | 查看学生档案 | studentId、warningId、returnContext |

### 7.2 出口

| 动作 | 去向 | 要求 |
|---|---|---|
| 查看档案 | 标准抽屉 | 打开正确学生 |
| 全屏查看 | 全屏档案模式 | 保留当前学生和记录位置 |
| 返回抽屉 | 标准抽屉 | 保留当前模块和记录位置 |
| 返回来源 | 原业务页面 | 恢复搜索、筛选、分页和抽屉上下文 |
| 查看来源记录 | 对应业务模块 | 保留返回档案入口 |

## 8. 学生档案列表

### 8.1 搜索
支持：
- 姓名模糊搜索
- 学号精确或前缀搜索

同名学生使用以下字段区分：
- 学号
- 年级
- 班级

### 8.2 快速筛选
- 年级
- 班级

规则：
- 年级和班级联动
- 班级选项只展示所选年级下的班级
- 返回列表后保留筛选条件

### 8.3 高级筛选
- 当前风险等级
- 当前处置状态
- 是否存在活动预警
- 是否有历史干预
- 负责心理老师
- 在校状态

说明：
- 负责心理老师可以作为筛选条件，但不作为列表字段
- 默认只显示在校学生
- 毕业或离校学生通过在校状态筛选进入

### 8.4 筛选逻辑
- 不同筛选类别之间为 AND
- 同一类别多选为 OR
- 清空高级筛选不清空搜索关键词
- “全部重置”同时清空搜索和筛选

### 8.5 列表字段

| 字段 | 内容 | 说明 |
|---|---|---|
| 学生信息 | 姓名、学号 | 用于身份确认 |
| 年级 / 班级 | 当前年级、当前班级 | 历史班级在详情中查看 |
| 操作 | 查看档案 | 打开标准抽屉 |

列表不展示：
- 风险等级
- 当前状态
- 最近测评
- 最近干预
- 负责心理老师
- 完整心理记录

## 9. 学生档案详情

### 9.1 学生信息
- 姓名
- 学号
- 当前年级 / 班级
- 当前班主任
- 负责心理老师；无负责人时显示“—”
- 在校状态
- 最近更新时间

暂不展示：
- 家长联系方式
- 家庭住址
- 与心理档案无关的学籍信息

### 9.2 当前心理状态概况
- 是否存在活动预警
- 当前风险等级
- 当前处置状态
- 最近测评时间
- 最近干预时间
- 已安排的下一事项，例如复测

规则：
- 当前风险和状态只来自活动事项
- 没有活动事项时显示“暂无活动事项”
- 最近一次闭环事项不能冒充当前状态
- 不展示系统生成的下一步专业建议

Phase 5.2A 数据规则：当前风险、当前状态、负责心理老师和事实提示均从共享 warning 活动事项派生；没有活动事项时显示“暂无活动事项”和“暂无活动风险”。

### 9.3 当前活动事项
存在活动事项时展示：
- 事项编号
- 来源
- 当前风险
- 当前状态
- 负责心理老师
- 最近动态
- 查看事项详情

不存在活动事项时展示明确空状态。

Phase 5.2A 额外展示事项开始时间，以及反馈、干预、复测和转介记录数量；本轮不提供“查看事项详情”按钮或无响应链接。

### 9.4 学籍动态
每条记录展示：
- 年级
- 班级
- 起始时间
- 结束时间
- 变动类型

规则：
- 当前阶段结束时间显示“至今”
- 转班和升年级不覆盖历史阶段
- 历史业务记录保留发生时年级和班级

### 9.5 业务事项分组
学生的业务记录必须按 `caseId` 或等价事项编号建立关联。

每个业务事项展示：
- 事项编号
- 起始时间
- 结束或闭环时间
- 来源
- 最高或确认风险等级
- 最终结果
- 关联测评数量
- 关联干预数量
- 关联复测数量

展开单个事项后，按时间展示：

```text
线索 / 测评
→ 正式预警
→ 干预
→ 复测
→ 继续干预 / 转介 / 闭环
```

这样用来解释多次干预与复测之间的关系。

### 9.6 测评记录
每条展示：
- 测评类型
- 完成时间
- 结果摘要
- 风险提示
- 数据来源
- 发生时年级 / 班级
- 所属业务事项
- 查看详情入口

### 9.7 预警记录
每条展示：
- 形成时间
- 线索来源
- 确认风险等级
- 处置状态
- 负责心理老师
- 结束或闭环结果
- 所属业务事项
- 查看事项详情

### 9.8 干预记录
每条展示：
- 干预时间
- 记录人
- 干预方式
- 情况摘要
- 本次判断
- 后续计划
- 所属业务事项

规则：
- 多次记录最新在前
- 不覆盖历史
- 可展开完整记录
- 不在档案页直接编辑

### 9.9 复测记录
每条展示：
- 安排时间
- 计划复测时间
- 实际完成时间
- 结果摘要
- 与上次结果对比
- 心理老师结论
- 所属业务事项

### 9.10 转介与闭环记录
展示：
- 发起转介时间
- 转介类型 / 机构
- 记录结果时间
- 结果摘要
- 是否安排后续复测
- 闭环时间
- 闭环说明
- 所属业务事项

### 9.11 全量业务时间线
- 聚合测评、预警、干预、复测、转介和闭环事件
- 最新在前
- 每条显示时间、事件、记录人、所属事项
- 只展示真实业务事件
- 查看详情、切换全屏等阅读行为不写入业务时间线

## 10. 数据与历史规则
1. 当前基础信息与历史阶段信息分开保存。
2. 转班、升年级和毕业不能覆盖历史班级。
3. 历史业务记录不能删除。
4. 档案页不直接修改来源业务记录。
5. 当前风险和状态只来自活动事项。
6. 最近一次闭环事项属于历史事项。
7. 所有记录按时间倒序。
8. 每条记录保留来源、发生时间、记录人和所属事项。
9. 同一业务事项中的测评、预警、干预、复测、转介和闭环使用同一 `caseId`。
10. 档案聚合数据应从来源模块派生，不建立可单独编辑的重复副本。
11. 学生基础 mock 只保存身份、学籍与真实 `warningCaseIds`；当前风险、状态、负责人和干预历史不得作为独立真值保存。
12. 当前 Demo 的 `caseId` 对应 `WarningItem.id`，关联 ID 必须使用真实 `WRN-*`。
13. 活动事项满足 `isActive=true` 且 `currentStatus!=closed`；关闭和结束线索处理的事项进入历史摘要。

## 11. 页面状态和异常

| 情况 | 页面处理 |
|---|---|
| 同名学生 | 使用学号、年级和班级区分 |
| 无历史记录 | 保留学生基础信息，各记录模块显示空状态 |
| 列表加载失败 | 保留页面框架并提供重试 |
| 单条记录失败 | 单条提示，不阻塞整个档案 |
| 已毕业 / 离校 | 通过在校状态筛选进入，历史记录保留 |
| 学生转班 | 当前班级与历史班级分开展示 |
| 从业务模块返回 | 恢复来源上下文 |
| 记录过多 | 模块折叠、事项分组、分页或全屏查看 |
| 全屏切换失败 | 保留标准抽屉，不丢失当前学生 |
| 某来源记录被撤销 | 展示来源状态，不从档案中静默消失 |

## 12. MVP 验收标准
1. 能按姓名和学号搜索。
2. 能通过年级和班级筛选。
3. 能通过高级筛选查找活动预警、历史干预、负责人和历史学生。
4. 同名学生不会误选。
5. 列表只展示学生信息、年级 / 班级和查看操作。
6. 能从其他业务模块直接打开正确学生档案。
7. 返回来源页面后上下文保留。
8. 标准抽屉宽度与预警详情抽屉统一。
9. 全屏模式与抽屉数据一致，但采用不同布局。
10. 能区分当前活动事项和历史事项。
11. 能按业务事项看出干预与复测关系。
12. 能查看学籍动态。
13. 能查看测评、预警、干预、复测、转介和闭环记录。
14. 多条记录最新在前。
15. 所有心理老师均可查看完整档案。
16. 基础信息只读。
17. 档案页不直接修改或删除历史记录。
18. 组织和人员 CRUD 不出现在本轮。
