# 系统设置 MVP 流程规格

## 页面进入

```text
点击系统设置
→ 读取共享学校、组织、教师、学生和管理记录
→ 默认学校信息
```

## 单条维护

```text
编辑
→ 本地校验
→ 提交共享 action
→ 校验 version 与引用
→ 原子替换基础数据
→ 写 AdminAuditLog
→ 下游重新派生
```

学校名称影响 Topbar；当前学期影响校级总览；学生当前资料影响学生档案和总览当前组织；教师影响当前负责人筛选及新反馈、新预约默认对象。任何当前资料更新都不改写历史业务快照。

## 学籍变更

```text
选择目标班级和生效日期
→ 校验目标组织
→ 关闭旧学籍区间
→ 新增学籍历史
→ 更新当前年级 / 班级
→ 写管理操作记录
```

毕业和离校只更新学籍状态与历史，不删除档案和事项。

## 教师停用

```text
检查活动预警负责人
+ 当前班主任
+ 未来干预预约负责人
→ 有引用则返回冲突
→ 无引用则停用
```

## 批量导入

```text
选择固定模板和模式
→ 解析
→ 规范化
→ 行级 / 跨行 / 引用校验
→ 生成 Preview
→ 无 Error 后确认
→ 校验 baseDataVersion
→ 在内存副本执行
→ 全部成功后替换
```

预览、取消和下载模板不写管理操作记录。确认成功或失败写独立导入审计，不写预警时间线。
