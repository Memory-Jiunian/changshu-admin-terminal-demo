# 工作台 MVP 验收

- [x] 只展示当前心理老师的活动事项
- [x] 六类待办和一类提醒按规则派生
- [x] 今日复测提醒不计入待办总数
- [x] 新反馈与反馈超时互斥
- [x] 今日复测提醒与复测结果待更新互斥
- [x] 转介跟进中持续存在且不标逾期
- [x] 稳定 ID、排序和数据异常可测试
- [x] 六个预警详情锚点存在并可定位
- [x] 目标模块缺失安全回退概况
- [x] 新反馈模块展示后保持未读，明确确认后才标记已读
- [x] 已读更新不改变反馈记录和业务时间线
- [x] 返回上下文恢复筛选、滚动和选中任务
- [x] 首次进入展示真实工作台
- [x] 未实现页面展示占位而非预警管理
- [x] 预警管理和学生档案回归通过

## Phase W1.1

- [x] 共享 mock 覆盖六类主动待办和一类今日提醒
- [x] 今日提醒显示“查看安排”，定位复测模块且只查看不消失
- [x] 任一反馈记录未读即派生有新反馈
- [x] 工作台和普通预警入口共用“标记为已查看”
- [x] 标记已读保留反馈正文、反馈记录和业务时间线
- [x] 工作台新反馈来源未读关闭时出现阻断式保护 Dialog
- [x] 关闭并保留待办不修改阅读状态
- [x] 标记已查看并关闭后任务消失
- [x] 普通预警入口和 warningId 不一致时不触发关闭保护

## Phase I1

- [x] 正式预警且无有效预约时派生“待安排干预”。
- [x] 待干预预约已过期但未处理时派生“干预情况待确认”。
- [x] 待复测计划已过期但未完成时派生“复测情况待确认”。
- [x] 今天和次日的干预预约显示在今日/近期安排，不计入主动待办总数。
- [x] 干预任务定位 `intervention`，复测任务定位 `retest`。
- [x] 记录结果、改约、取消或完成复测后，工作台根据共享 warning 自动重新派生。
- [x] 派生任务不建立独立完成状态。

## Phase I1.1

- [ ] 工作台不再显示“干预情况待确认”和“复测情况待确认”任务 Tab。
- [ ] 两类待确认进入今日 / 近期安排，并使用“确认干预情况”或“查看并重新安排”。
- [ ] 干预 60 分钟、复测 120 分钟宽限期边界正确，时间经过不修改共享 warning。
- [ ] 当前待办包含需要主动确认的安排项；今日提醒只统计 upcoming 安排。
- [ ] 工作台只统计当前心理老师负责事项，待安排干预数量与本人符合条件的正式预警一致。
