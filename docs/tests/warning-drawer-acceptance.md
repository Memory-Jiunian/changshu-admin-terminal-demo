# 学生风险详情抽屉验收基线

## 1. 使用说明

本文档用于验收预警管理详情抽屉。所有记录只展示已经发生的事实，来源路径不同，前置事件不同，不给所有学生强行补全完整流程。

待复核事项必须已经完成必要补充评估并形成风险依据。刚进入线索池的事项不属于 `pending_review`，也不应出现在当前预警管理 Demo。

## 2. 全局数据语义

- [ ] `sourceType` 和 `evidenceTypes` 分开建模。
- [ ] 来源只允许普筛异常、班主任上报、AI倾诉主动触发。
- [ ] 风险依据只允许班主任上报、AI倾诉、深度测评。
- [ ] `sourceType` 和 `evidenceTypes` 保留在数据模型中，但当前风险依据正文不直接展示来源或依据类型标签列表。
- [ ] 普筛异常只可用于来源语义和时间线，不作为风险依据。
- [ ] `suggestedRiskLevel` 和 `confirmedRiskLevel` 分开建模。
- [ ] 有效展示等级为 `confirmedRiskLevel ?? suggestedRiskLevel`。
- [ ] 只有心理老师确认正式预警后，`confirmedRiskLevel` 才有值。
- [ ] `confirmedRiskLevel` 与 `suggestedRiskLevel` 不同时存在非空调整理由。
- [ ] 学生信息卡不展示风险等级标签，仍展示当前状态标签。
- [ ] 风险依据标题右侧展示当前有效风险等级；已确认时显示 `confirmedRiskLevel`，未确认时显示 `suggestedRiskLevel`。
- [ ] 风险依据正文不展示“来源”字段或 `evidenceTypes` 标签列表。
- [ ] 系统提示等级、心理老师确认等级和风险等级调整理由仍正常展示。

## 3. 待复核

- [ ] 当前状态显示“待复核”。
- [ ] 事项时间线已包含“发起补充评估”和“补充评估完成”。
- [ ] 不存在只入线索池、尚未完成补充评估的待复核样例。
- [ ] 展示学生概况、风险依据、处置时间线和底部操作区。
- [ ] 不展示班主任反馈、干预记录或复测记录模块。
- [ ] 底部按钮为“结束本次线索处理 / 继续观察 / 确认正式预警”。
- [ ] `confirmedRiskLevel` 为空，显示“待心理老师确认”。
- [ ] 时间线不出现独立“心理老师复核”。
- [ ] 禁止出现确认正式预警、干预、复测、转介或闭环事件。

## 4. 观察中

- [ ] 当前状态显示“观察中”。
- [ ] 展示学生概况、风险依据、处置时间线和底部操作区。
- [ ] 不展示班主任反馈、干预记录或复测记录模块。
- [ ] 底部按钮为“继续观察 / 确认正式预警 / 结束本次线索处理”。
- [ ] 时间线出现“发起补充评估 → 补充评估完成 → 标记继续观察”。
- [ ] `confirmedRiskLevel` 为空，显示“待心理老师确认”。
- [ ] 时间线不出现独立“心理老师复核”。
- [ ] 禁止出现确认正式预警、干预、复测、转介或闭环事件。

## 5. 正式预警

- [ ] 当前状态显示“正式预警”。
- [ ] 展示学生概况、风险依据、班主任反馈、处置时间线和底部操作区。
- [ ] 底部按钮为“请求补充反馈 / 记录干预”。
- [ ] `confirmedRiskLevel` 有值并作为有效展示风险等级。
- [ ] 确认正式预警时间线描述包含心理老师确认的正式风险等级。
- [ ] 确认正式预警时间线描述包含“系统已同步生成班主任协作任务”。
- [ ] 等级被调整时，确认时间线和风险依据显示调整理由。
- [ ] 时间线不出现独立“心理老师复核”或“系统同步班主任协作任务”。
- [ ] 请求补充反馈对象只能是班主任。
- [ ] 没有反馈记录时不显示“查看完整反馈”。

## 6. 干预中

- [ ] 当前状态显示“干预中”。
- [ ] 展示学生概况、风险依据、班主任反馈、干预记录、处置时间线和底部操作区。
- [ ] 底部按钮为“新增干预记录 / 安排复测 / 转介”。
- [ ] 干预记录展示干预时间、记录人、干预方式、情况摘要、本次判断和后续计划。
- [ ] 多条干预记录按 `occurredAt` 倒序展示，最新在顶部。
- [ ] 时间线至少包含“确认正式预警”和已发生的“新增干预记录”。
- [ ] 未安排复测时，禁止出现复测提醒、复测完成或更新状态。
- [ ] 未发起转介时，禁止出现记录转介结果。

## 7. 待复测

- [ ] 当前状态显示“待复测”。
- [ ] 展示学生概况、风险依据、班主任反馈、干预记录、复测记录、处置时间线和底部操作区。
- [ ] 底部按钮为“查看复测结果 / 更新状态”。
- [ ] “更新状态”只提供“风险解除并闭环 / 继续干预 / 转介”。
- [ ] 复测记录展示安排时间、计划复测时间、实际完成时间、结果摘要、与上次结果对比和心理老师结论。
- [ ] 多条复测记录按安排时间倒序展示。
- [ ] 时间线出现“安排复测”和“班主任待复测提醒”。
- [ ] 学生实际完成后才出现“学生完成复测”。
- [ ] 点击“查看复测结果”不新增业务时间线。
- [ ] 时间线不显示“查看复测结果”。

## 8. 转介中

- [ ] 当前状态显示“转介中”。
- [ ] 展示学生概况、风险依据、班主任反馈、干预记录、已有复测记录、处置时间线和底部操作区。
- [ ] 底部按钮为“记录转介结果 / 安排复测”。
- [ ] 时间线至少出现“发起转介”。
- [ ] 只有实际记录外部反馈后才出现“记录转介结果”。
- [ ] 记录转介结果后主状态仍为“转介中”。
- [ ] 记录转介结果不会自动生成复测记录或复测时间线。
- [ ] 只有执行“安排复测”后才能进入待复测。
- [ ] 发起转介后通知对应班主任，年级主任不接收该通知。

## 9. 已闭环

- [ ] 当前状态显示“已闭环”。
- [ ] 展示学生概况、风险依据、班主任反馈、干预记录、复测记录、处置时间线和底部操作区。
- [ ] 底部按钮只显示“查看归档记录”。
- [ ] 时间线出现形成闭环所依据的“更新复测状态”和“完成闭环归档”。
- [ ] 时间线不显示“查看复测结果”。
- [ ] 复测记录保留结果摘要、比较和心理老师结论。
- [ ] 完成闭环后通知对应班主任，年级主任不接收该通知。
- [ ] 查看归档记录不新增业务时间线，不修改主处置状态。

## 10. 确认正式预警交互

- [ ] 待复核和观察中的“确认正式预警”会打开确认弹窗。
- [ ] 弹窗展示学生姓名和系统提示风险等级。
- [ ] 正式风险等级必选，选项为中风险、高风险、危险。
- [ ] 正式等级与系统提示不同时，调整理由必填。
- [ ] 正式等级与系统提示相同时，判断说明可选填。
- [ ] 校验未通过时不能提交。
- [ ] 确认后状态变为正式预警，反馈状态变为待反馈，未读反馈为 false。
- [ ] 确认后列表和抽屉同步显示有效风险等级、当前状态和最新动态。
- [ ] 只追加一条“确认正式预警”时间线，描述合并确认等级、调整理由和班主任协作任务。
- [ ] Phase 4 操作均通过类型化 action 打开相应 Dialog，并同步更新列表、数量、筛选结果、抽屉和全屏详情。

## 11. 记录排序与反馈

- [ ] 业务时间线按 `occurredAt` 倒序展示，最新在顶部。
- [ ] 时间线不单独展示“心理老师复核”。
- [ ] 时间线不展示“查看复测结果”。
- [ ] 班主任反馈按 `submittedAt` 倒序展示。
- [ ] 干预记录按 `occurredAt` 倒序展示。
- [ ] 复测记录按 `arrangedAt` 倒序展示。
- [ ] 有反馈不自动等于“有新反馈”。
- [ ] 只有 `hasUnreadFeedback=true` 时显示“有新反馈”。
- [ ] 无反馈时不显示“查看完整反馈”。

## 12. 列表、高级筛选与范围

- [ ] 默认列表仍只有学生信息、风险等级、当前状态、最新动态、发生时间、反馈状态和操作。
- [ ] 默认列表不展示来源、风险依据类型、下一步动作、负责心理老师或完整业务详情。
- [ ] 高级筛选“线索类型”只保留班主任上报、AI倾诉、深度测评。
- [ ] 高级筛选布局和多条件筛选主交互没有被重做。
- [ ] 项目中没有新增线索池页面。
- [ ] 抽屉关闭后，列表筛选状态保持不变。
- [ ] 抽屉宽度来自公共详情配置，未在 `StudentRiskDrawer` 中重复硬编码。
- [ ] 全屏详情与抽屉复用同一详情内容和同一 warning 数据。
- [ ] 全屏操作成功后抽屉、列表与状态数量同步更新。
- [ ] 切换全屏、查看复测结果和查看归档记录不写入业务时间线。

## 13. Phase 4.6 反馈、复测与归档验收

- [ ] 确认正式预警必须填写反馈要求和晚于确认时间的截止时间。
- [ ] 确认后只新增一条合并风险等级、反馈要求、截止时间和班主任协作任务的时间线。
- [ ] 反馈模块展示班主任姓名、脱敏手机号、任务截止时间和有效反馈状态。
- [ ] 反馈模块不展示“提醒班主任反馈”或其他重复重发入口。
- [ ] 任务进行中显示待反馈且底部为禁用的“等待反馈”。
- [ ] 无反馈且超过截止时间显示反馈超时，底部提供“重新请求反馈”。
- [ ] 已收到反馈后不显示请求或重新请求反馈。
- [ ] 列表、快捷筛选、高级筛选、反馈模块和底部操作使用同一有效反馈状态。
- [ ] 复测记录展示一项或多项量表名称。
- [ ] 查看复测结果弹窗不展示心理老师结论，但保留客观结果和完整记录入口。
- [ ] 完整复测记录展示学生、事项编号、安排/计划/完成时间、量表、结果、对比和补充说明。
- [ ] 归档展示完整事项概况、依据、反馈请求、反馈、干预、复测、转介和倒序时间线。

## 14. Phase 5.2B.1 共享完整记录

- [ ] 风险依据展示完整结构化测评和 AI 可见会话，不展示系统提示或隐藏信息
- [ ] 班主任反馈按请求轮次展示，一轮允许多次回复，主动补充单独展示
- [ ] 每条班主任反馈进入有效处置时间线且不重复
- [ ] 转介中按钮为“新增转介跟进 / 安排复测”，可连续追加跟进
- [ ] 新增跟进后仍为转介中，不自动复测或闭环
- [ ] 学生风险详情、学生档案和归档读取同一完整证据、协作和转介数据
- [ ] 归档展示所有转介跟进及有效时间线，查看行为仍不写时间线
- [ ] 完整复测记录与归档均只读，查看行为不改变状态或时间线。

## 15. 工作台定向定位兼容

- [ ] 概况、风险依据、反馈、复测、转介和操作区存在稳定锚点。
- [ ] 工作台意图只定位正确 warningId 的目标模块。
- [ ] 普通预警入口仅在点击“标记为已查看”后更新反馈阅读状态。
- [ ] 已读更新不新增学生处置时间线。
- [ ] 反馈模块渲染、滚动和高亮均不会自动标记已读。
