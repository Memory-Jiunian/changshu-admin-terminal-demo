# 场景化走查卡：学生心理档案（优化版）

> 本文档将“列表页”和“详情抽屉 / 全屏模式”分开描述，避免页面范围混淆。

---

## 场景 1：已知姓名，查找学生历史

### 当前角色
心理老师。

### 触发场景
心理老师在新的预警或干预前，想确认某学生过去是否做过测评、干预或复测。

### 列表页应展示
- 姓名 / 学号搜索框
- 学生姓名
- 学号
- 当前年级 / 班级
- 查看档案操作

### 详情中应展示
- 学生基础信息
- 当前心理状态概况
- 当前活动事项
- 历史测评、预警、干预、复测、转介和闭环记录

### 用户应该可以做
- 模糊搜索姓名
- 用学号精确或前缀搜索
- 用年级 / 班级缩小范围
- 用学号、年级和班级区分同名学生
- 打开学生档案详情

### 用户不应该做
- 在列表查看完整测评、AI 倾诉或干预内容
- 仅凭姓名进入无法区分的同名学生
- 在列表直接修改专业记录

### 已确认结论
- 同名学生通过学号、年级和班级区分
- 返回列表后搜索和筛选条件必须保留
- 列表只展示基础定位信息

## 场景 2：只知道年级和班级

### 当前角色
心理老师。

### 触发场景
心理老师只知道某班学生过去接受过干预，需要从班级范围中定位。

### 列表页应展示
- 年级筛选
- 班级筛选
- 结果数量
- 学生基础列表

### 高级筛选可包含
- 当前风险等级
- 当前处置状态
- 是否存在活动预警
- 是否有历史干预
- 负责心理老师
- 在校状态

### 用户应该可以做
- 先选年级，再选对应班级
- 在筛选结果中继续搜索
- 按更新时间排序
- 清空筛选
- 筛选有历史干预的学生

### 已确认结论
- 年级和班级联动
- 当前班级与历史班级分开保存
- 负责心理老师不作为列表字段，但允许作为高级筛选条件
- “有历史干预”作为高级筛选项

## 场景 3：从预警管理进入学生档案

### 当前角色
正在处理预警的心理老师。

### 触发场景
心理老师在预警详情中需要查看学生过往测评、预警和干预记录。

### 页面应展示
- 正确学生的档案详情
- 当前活动事项
- 当前风险和状态
- 历史测评、预警、干预和复测记录
- 返回原预警详情入口

### 用户应该可以做
- 不经二次搜索直接进入档案
- 快速定位与当前事项相关的历史记录
- 返回原预警页面继续处理

### 用户不应该做
- 因跳转丢失原页面筛选、分页和抽屉上下文
- 在档案中直接改变预警状态
- 用历史风险覆盖当前风险

### 已确认结论
- 当前事项和历史事项必须清晰区分
- 返回后原上下文必须保留
- 档案提供历史依据，不重复预警抽屉的完整交互

## 场景 4：第二次或第三次干预前查看历史

### 当前角色
心理老师。

### 触发场景
学生再次需要干预，心理老师想先看前几次做了什么、效果如何。

### 详情应展示
- 按业务事项分组的记录
- 干预时间
- 记录人
- 干预方式
- 情况摘要
- 本次判断
- 后续计划
- 对应复测结果
- 对应转介或闭环结果

### 用户应该可以做
- 按最新在前查看多次干预
- 展开单次完整记录
- 查看同一事项下的干预与复测关系
- 在内容较多时切换全屏

### 用户不应该做
- 在档案页直接修改旧干预记录
- 用最新记录覆盖历史记录
- 把全局时间线摘要当作完整干预记录

### 关系展示规则
所有专业记录通过 `caseId` 或等价的业务事项编号关联。

单个事项内部建议按以下顺序展示：

```text
风险线索 / 测评
→ 正式预警
→ 第一次干预
→ 复测
→ 继续干预 / 转介 / 闭环
```

这样可以看出“干预—复测—继续处理”的关系，而不是只看到互不关联的记录。

## 场景 5：学生升年级、转班或毕业

### 当前角色
心理老师。

### 触发场景
学生基础信息发生变化，但历史心理记录必须保留。

### 页面应展示
- 当前年级 / 班级
- 当前在校状态
- 学籍动态
- 历史记录发生时的年级 / 班级

### 学籍动态字段
- 年级
- 班级
- 起始时间
- 结束时间
- 变动类型，例如升年级、转班、毕业、离校

### 用户应该可以做
- 查看当前基础信息
- 查看历史阶段所在班级
- 通过在校状态入口查看毕业或离校学生

### 用户不应该做
- 删除历史心理档案
- 因转班覆盖历史记录中的原班级
- 直接维护学校组织和人员账号

### 已确认结论
- 学生列表默认展示在校生
- 通过“在校状态”筛选查看毕业或离校学生
- 历史班级采用“年级班级 + 起始时间—结束时间”展示

## 场景 6：抽屉内容过多，切换全屏

### 当前角色
心理老师。

### 触发场景
学生有多次测评、预警、干预和复测，标准抽屉不适合连续阅读。

### 标准抽屉
- 宽度与预警管理抽屉一致
- 适合快速查看
- 单列卡片
- 模块可折叠
- 固定全屏按钮

### 全屏模式
- 使用同一份学生档案数据
- 不重新加载为另一份档案
- 使用更适合长内容的布局
- 可采用“左侧学生概况 + 右侧记录内容”或两列布局
- 支持切回标准抽屉
- 保留当前展开模块和滚动定位

### 已确认结论
- 抽屉宽度可以修改
- 全部详情抽屉应引用统一宽度配置
- 全屏模式允许使用不同信息排版
- 不复制两套独立业务组件，优先复用同一数据和记录组件
