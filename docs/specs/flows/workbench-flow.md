# 工作台流程规格

共享 warnings → 当前老师与活动事项准入 → 六类待办和今日提醒派生 → 互斥与排序 → 页面展示。

任务点击 → 保存 `WorkbenchReturnContext` → 打开正确预警详情 → 等待目标锚点渲染 → 定位并短暂高亮。

新反馈任务只负责打开正确事项并定位反馈模块。反馈模块渲染后保持未读；心理老师明确点击“标记为已查看”时，统一 action 写入未读反馈记录的 `psychologistReadAt` 并同步兼容字段。该阅读更新不写时间线。

工作台新反馈来源且仍有未读反馈时，关闭详情先确认：继续查看、关闭并保留待办，或标记已查看并关闭。普通预警入口不启用关闭保护。

返回工作台 → 恢复筛选和滚动 → 重新派生；已失效任务不恢复，其他任务保留。

目标模块不存在时打开最新事项概况并提示状态已更新；warningId 不存在时保留工作台并提示事项不存在。

## Phase I1 派生规则

工作台主动任务继续包含既有六类任务和“待安排干预”。“干预情况待确认”和“复测情况待确认”不再作为任务 Tab，而是进入今日 / 近期安排。

安排项分为 `upcoming`、`intervention_confirmation_required` 和 `retest_incomplete`。需要确认优先，其次未完成，再按今天、次日和计划时间排序。干预预约超过 60 分钟宽限期后显示“预约时间已过 · 待确认”；复测超过 120 分钟宽限期后显示“复测未完成”。时间经过不修改业务记录。

当前待办统计普通任务和需要主动确认的安排项；今日提醒只统计 upcoming 安排；逾期事项只统计具有截止时间且确已超时的任务。所有任务和安排只纳入当前心理老师负责事项。待安排干预数量必须等于本人正式预警且无有效预约的事项数量。
