# 流程规格：学生心理档案查询与详情 MVP

## 1. 文档目的
定义学生心理档案列表、详情抽屉、全屏模式、跨模块进入、记录聚合和历史保存规则。

本模块只负责查询与聚合，不负责产生或修改测评、预警、干预、复测、转介和闭环记录。

Phase 5.2A 只实现共享 warning 数据源和核心详情摘要；全屏、完整专业记录及跨模块跳转留待后续阶段。

Phase 5.2A.1 例外开放学生档案查看对应预警详情并返回原上下文；其他跨模块路径仍不实现。

## 2. 领域对象

### 2.1 StudentProfile
学生当前基础信息：
- studentId
- studentName
- studentNumber
- currentGrade
- currentClass
- currentHeadTeacher
- currentResponsiblePsychologist
- enrollmentStatus
- updatedAt

### 2.2 EnrollmentHistory
学生学籍动态：
- id
- studentId
- grade
- className
- changeType
- startedAt
- endedAt

### 2.3 MentalHealthCase
一次连续的心理健康处置事项：
- caseId（当前 Demo 对应 `WarningItem.id`）
- studentId
- sourceType
- startedAt
- closedAt
- currentStatus
- confirmedRiskLevel
- finalOutcome
- responsiblePsychologist

### 2.4 业务记录
所有业务记录必须关联：
- studentId
- caseId
- recordId
- occurredAt
- recordedBy
- sourceModule

记录类型包括：
- AssessmentRecord
- WarningRecord
- InterventionRecord
- RetestRecord
- ReferralRecord
- ClosureRecord
- TimelineEvent

## 3. 核心业务不变量
1. 学生档案只聚合来源业务记录，不生成独立可编辑副本。
2. 学生档案不能直接修改或删除来源业务记录。
3. 当前风险和当前状态只来自未结束的活动事项。
4. 没有活动事项时显示“暂无活动事项”。
5. 历史闭环状态不能作为当前状态展示。
6. 同一事项中的测评、预警、干预、复测、转介和闭环使用同一 `caseId`。
7. 多个不同事项不得混合为一条连续处置记录。
8. 当前班级和历史发生时班级分开保存。
9. 历史记录统一按时间倒序展示。
10. 所有心理老师在当前 MVP 中均可查看完整学生档案。
11. 标准抽屉和全屏模式读取同一数据源。
12. 查看档案、展开模块和切换全屏不写入业务时间线。
13. 学生基础数据不保存当前风险、当前状态、负责心理老师或干预历史；这些字段从共享 warning 数据派生。
14. 页面切换不得重置共享 warning 数据。
15. `warningCaseIds` 必须匹配真实 `WarningItem.id`。

## 4. 查询与筛选流程

```text
读取有效的最近班级偏好，或选择第一个有学生的班级
→ 无关键词：按年级 Tab + 班级 Tab 浏览
→ 有姓名/学号关键词：忽略班级 Tab，全校搜索
→ 高级筛选始终参与过滤
→ 每页最多 30 人
```

最近班级只在隔离的 localStorage key 中保存年级和班级。搜索、筛选、页码、学生身份和档案内容只进入临时 returnContext。

筛选读取范围：当前风险、预警状态、当前负责人读取活动事项；事项来源、正式预警确认和干预记录读取全部关联事项。

### 4.1 导航进入

```text
心理老师点击“学生档案”
→ 加载默认学生列表
→ 默认只显示在校学生
→ 心理老师搜索或筛选
→ 查看学生档案
```

### 4.2 搜索流程

```text
输入姓名或学号
→ 系统在当前筛选范围内匹配
→ 返回学生基础列表
→ 同名学生使用学号、年级和班级区分
```

规则：
- 姓名支持模糊搜索
- 学号支持精确或前缀搜索
- 搜索不自动清空已有筛选
- 清空关键词后恢复当前筛选结果

### 4.3 年级 / 班级筛选

```text
选择年级
→ 班级选项更新为该年级下的班级
→ 选择班级
→ 列表刷新
```

规则：
- 年级和班级联动
- 清除年级时同步清除无效班级
- 返回列表时保留条件

### 4.4 高级筛选
支持：
- 当前风险等级
- 当前处置状态
- 是否存在活动预警
- 是否有历史干预
- 负责心理老师
- 在校状态

规则：
- 不同类别之间为 AND
- 同一类别多选为 OR
- 负责心理老师不必展示为列表列
- 默认在校状态为“在校”
- “全部重置”才清空关键词和全部筛选

## 5. 打开学生档案流程

```text
点击“查看档案”
→ 系统根据 studentId 加载档案聚合数据
→ 打开标准详情抽屉
→ 默认展示学生信息、当前概况和当前活动事项
→ 其他历史模块按需渲染
```

加载内容：
1. StudentProfile
2. 当前活动事项
3. EnrollmentHistory
4. 历史 MentalHealthCase
5. 各事项关联业务记录
6. 全量业务时间线

异常：
- 学生基础信息加载失败：不打开空抽屉，提示重试
- 单个历史模块失败：保留其他模块并显示局部错误
- 无历史记录：展示空状态，不隐藏基础信息

## 6. 从其他模块进入

### 6.1 从预警管理进入

```text
预警详情点击“查看学生档案”
→ 带入 studentId、warningId、returnContext
→ 直接打开该学生档案抽屉
→ 优先定位 warningId 所属事项
```

### 6.2 从干预记录进入

```text
干预记录点击“查看学生档案”
→ 带入 studentId、interventionId、returnContext
→ 打开档案
→ 优先定位 interventionId 所属事项
```

### 6.3 返回来源

```text
关闭学生档案
→ 根据 returnContext 返回来源页面
→ 恢复来源页面的筛选、分页、抽屉和选中数据
```

档案模块不得破坏来源页面状态。

## 7. 标准抽屉与全屏流程

### 7.1 显示状态

```text
closed
→ drawer
→ fullscreen
→ drawer
→ closed
```

### 7.2 标准抽屉
- 使用公共详情抽屉宽度
- 与预警详情抽屉一致
- 单列结构
- 支持模块折叠
- 适合快速浏览

### 7.3 全屏模式

```text
在抽屉点击“全屏查看”
→ 保留当前 studentId
→ 保留已展开模块和记录定位
→ 切换为全屏布局
```

全屏模式：
- 不创建第二份档案数据
- 不重复请求已加载的数据，除非数据已失效
- 使用同一业务记录组件
- 允许两列或主从布局
- 支持切回抽屉

### 7.4 统一宽度规则
预警详情抽屉、学生档案抽屉及后续同级详情抽屉必须引用公共宽度配置。

禁止：
- 各组件分别硬编码不同宽度
- 修改一个抽屉时遗漏其他同级抽屉
- 为全屏模式复制完整业务逻辑

## 8. 当前事项与历史事项

### 8.1 当前活动事项
满足以下条件的事项视为活动事项：
- `isActive=true`
- `currentStatus!=closed`

当前假设每名学生最多一条活动事项。若聚合发现多条，返回数据异常并临时按 `activityTime` 最新项展示；正式 mock 不得包含该冲突。
- 状态属于当前业务定义中的活动状态

一个学生如果存在活动事项，详情概况展示该事项的：
- caseId
- 当前风险等级
- 当前状态
- 负责心理老师
- 最近动态

### 8.2 无活动事项
没有活动事项时：
- 显示“暂无活动事项”
- 当前风险显示“—”或“暂无活动风险”
- 不使用最近一次闭环风险和状态代替当前值

### 8.3 历史事项
`currentStatus=closed` 或 `isActive=false` 的事项进入历史事项列表。

每条历史事项展示：
- 起止时间
- 来源
- 确认风险
- 最终结果
- 关联记录数量

## 9. 业务事项关联流程

所有记录通过 `caseId` 聚合。

当前 Demo 不新增独立 `caseId` 字段，直接使用 `WarningItem.id`。

## 8.4 共享状态流程

```text
AdminDataProvider 初始化 warningMockData
→ 预警管理读取并更新共享 warnings
→ 页面切换不卸载 Provider
→ 学生档案从同一 warnings 派生摘要和详情
→ 返回预警管理时保留更新后的数据
```

## 8.5 有限跨模块返回流程

```text
学生档案抽屉点击“查看预警详情”
→ 组件仅提交 warningId
→ App 保存类型化 returnContext
→ 预警管理打开对应 warningId
→ 点击返回学生档案
→ 恢复搜索、Tab、筛选、页码、选中学生、抽屉和滚动位置
```

## 8.6 Phase 5.2B 事项详情流程

```text
学生档案概览
→ 当前或历史事项点击“查看完整记录”
→ 同一抽屉切换到 case_detail(warningId)
→ 只渲染该事项的共享只读记录
→ 返回学生档案并恢复概览滚动位置
```

若从事项详情点击“查看预警详情”，App returnContext 还需保存 `drawerView`、`selectedCaseId`、`profileScrollTop`、`caseDetailScrollTop` 和 `expandedRecordSections`。事项已不存在或与学生不匹配时安全退回概览并提示数据已更新。

事项记录排序：反馈请求按 `requestedAt`、反馈按 `submittedAt`、干预按 `occurredAt`、复测按 `arrangedAt`、转介按 `referredAt`、时间线按 `occurredAt` 倒序。排序只作用于派生副本。

## 8.7 Phase 5.2B.1 记录与导出流程

```text
结构化测评 / AI 可见会话 → WarningItem 完整证据数组 → 预警、档案、归档、导出共享展示
反馈请求 + requestId 回复 → 协作轮次 → 派生反馈时间线
转介中 → 新增转介跟进（可重复）→ 仍为转介中
学生档案 → 导出设置 → 结构化打印报告 → 浏览器打印 / 保存 PDF → 恢复原上下文
```

导出当前事项仅在 `case_detail` 可选；敏感完整记录默认关闭。浏览、切换、打印和取消打印均不写时间线。

### 9.1 单个事项展示顺序
按业务含义展示：

```text
风险线索 / 测评
→ 正式预警
→ 干预
→ 复测
→ 继续干预 / 转介 / 闭环
```

### 9.2 多次干预
每次干预保留独立记录。

```text
第一次干预
→ 第一次复测
→ 第二次干预
→ 第二次复测
→ 最终结果
```

不得：
- 用最后一次干预覆盖前次记录
- 将不同事项的复测关联到当前事项
- 只按记录类型分组而丢失事项关系

### 9.3 复测关系
复测记录必须关联：
- 所属 caseId
- 触发该复测的干预或转介记录
- 上一次可对比测评
- 复测后的心理老师结论

## 10. 学籍动态流程

### 10.1 学生转班 / 升年级

```text
接收新的当前年级或班级
→ 关闭上一条 EnrollmentHistory
→ 写入 endedAt
→ 新增一条 EnrollmentHistory
→ 更新 StudentProfile 当前年级 / 班级
```

规则：
- 历史记录中的发生时班级不更新
- 当前阶段结束时间显示“至今”
- 转班和升年级都保留历史阶段

### 10.2 毕业 / 离校

```text
学生状态更新为毕业或离校
→ 关闭当前 EnrollmentHistory
→ 默认学生列表不再展示
→ 可通过“在校状态”筛选查看
→ 心理健康历史继续保留
```

## 11. 全量业务时间线
时间线聚合：
- 测评完成
- 形成预警
- 干预记录
- 安排和完成复测
- 发起和记录转介
- 完成闭环

规则：
- 最新在前
- 每条包含事件、时间、记录人和 caseId
- 只记录业务事件
- 不记录打开档案、查看详情、切换全屏和展开模块
- 某条来源记录被撤销时，展示其状态变化，不静默删除历史

## 12. 页面操作边界
学生档案允许：
- 搜索
- 筛选
- 查看
- 展开和折叠
- 切换全屏
- 跳转来源业务记录
- 返回来源页面

学生档案禁止：
- 修改学生心理记录
- 删除历史业务记录
- 改变预警或干预状态
- 新增干预、复测或转介
- 修改确认风险等级
- 维护学校组织和人员账号

## 14. 上下游影响

### 上游
- 工作台需要提供 studentId
- 预警管理需要提供 studentId、warningId 和 returnContext
- 干预记录需要提供 studentId、interventionId 和 returnContext

### 下游
- 学生档案中的来源记录入口需要能返回对应业务模块
- 预警、干预、复测、转介和闭环数据模型需要包含 studentId 和 caseId
- 学籍系统或 mock 数据需要提供 EnrollmentHistory
- 公共详情抽屉需要提供统一宽度配置和全屏显示能力

## 16. MVP 验收规则
1. 默认只展示在校学生。
2. 姓名、学号、年级和班级搜索筛选正确。
3. 同名学生不会误选。
4. 列表只显示基础定位字段。
5. 高级筛选可以使用未展示为列表列的数据。
6. 查看档案打开标准抽屉。
7. 抽屉宽度与预警详情统一。
8. 全屏模式复用同一数据，并使用不同布局。
9. 返回列表和来源模块时上下文保留。
10. 当前活动事项和历史事项正确区分。
11. 没有活动事项时不显示历史闭环状态作为当前状态。
12. 同一 caseId 下的干预与复测关系可被识别。
13. 多次记录不会覆盖。
14. 学籍动态保留历史班级和时间。
15. 毕业或离校学生可通过筛选查看。
16. 档案页不修改任何来源业务记录。

## Phase I1 记录同步

学生档案继续按 `WarningItem.id` 聚合只读事项记录。干预模块同时展示预约历史和实际干预记录；预约状态、改约链路和通知计划均来自共享 warning。复测记录通过 `assessmentRecordId` 关联完整测评作答，不复制测评原始数据。转介跟进分别展示事实摘要和专业结论。档案查看、归档和导出均不得修改预约状态、主状态或时间线。
