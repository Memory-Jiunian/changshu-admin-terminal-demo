# 预警管理领域规格

## 1. 文档目的

本文档定义预警管理的稳定领域语言、角色边界、数据语义和业务不变量。页面行为以 `PRD.md` 为依据，状态流转以 `docs/specs/flows/warning-management-flow.md` 为依据；发生冲突时按 `AGENTS.md` 的规格优先级处理。

当前实现范围以 `TASKS.md` 中的 Active scope 为准，不在领域规格中长期写死。

## 2. 项目目标

管理终端接收并汇集由普筛、AI倾诉、深度测评和班主任上报产生的风险线索，由心理老师完成专业复核、正式预警确认、干预、复测、转介和闭环管理。

系统只负责汇集事实、呈现状态、记录过程和提供操作入口，不替代心理老师做专业心理判断。

## 3. 三端分工

| 终端 | 主要职责 | 不承担的职责 |
|---|---|---|
| 学生端 | 完成普筛、深度测评和 AI 倾诉，产生测评结果和风险线索 | 不展示预警处置流程，不进行专业风险定级 |
| 小程序端 | 向班主任触达协作任务和复测提醒；向年级主任展示协作进度和督办事项 | 不确认正式预警，不修改风险等级或主处置状态，不展示完整测评和完整 AI 对话 |
| 管理终端 | 汇集上游线索、证据和协作反馈；由心理老师完成复核、干预、复测、转介和闭环管理 | 不作为学生测评入口，不替代小程序做移动协作，不替代专业判断 |

## 4. 角色与权限边界

### 4.1 心理老师

负责心理老师可以查看和操作完整预警详情，包括证据摘要、班主任反馈、干预记录、复测记录和处置时间线。

其他心理老师默认只能查看：

- 事项是否已有负责老师
- 事项是否正在处理中
- 负责心理老师是谁

其他心理老师只有在以下情况下可查看完整详情：

- 事项转交
- 被邀请协作
- 值班应急接管
- 心理中心负责人授权

跨负责人的查看和操作必须留痕。本轮只定义权限边界，不实现真实权限系统。

### 4.2 班主任

班主任通过小程序：

- 上报异常
- 提交事实观察
- 提交协作反馈
- 提醒学生按时复测

班主任不得判断风险等级、确认正式预警、修改主处置状态、查看完整测评或完整 AI 对话。

### 4.3 年级主任

年级主任通过小程序：

- 查看协作进度
- 发送督办提醒
- 留下督办记录

年级主任不得提交学生事实观察或班主任协作反馈，不得判断风险等级、确认正式预警、修改主处置状态、查看完整测评或完整 AI 对话。

### 4.4 校级管理者

校级管理者只查看脱敏后的学校整体趋势、处置进度和资源调度信息，不直接进入学生个人心理档案，不查看完整学生风险详情。

## 5. 线索池与待复核边界

线索池是预警管理的上游业务容器，承载：

- 普筛异常
- 班主任上报异常
- AI倾诉主动触发风险

线索池中的事项尚未进入预警管理主状态。线索池进度可以包括：

- 待发起补充评估
- 补充评估中
- 补充评估已完成
- 已转入预警复核
- 无需继续处理

待复核的定义是：学生已经完成必要的补充评估，系统已经形成可供心理老师判断的风险依据，等待心理老师进行专业复核。

“进入线索池”不等于进入 `pending_review`。当前 Demo 不实现线索池页面，预警管理 Demo 从“补充评估完成并形成待复核事项”开始。

## 6. 主处置状态

主处置状态与反馈状态是两个独立维度，不得互相替代。

| 代码 | 中文名称 | 含义 |
|---|---|---|
| `pending_review` | 待复核 | 补充评估已完成并形成风险依据，等待心理老师提交复核结果 |
| `observing` | 观察中 | 心理老师已提交继续观察的复核结果 |
| `formal_warning` | 正式预警 | 心理老师已确认正式风险等级并形成正式预警 |
| `in_intervention` | 干预中 | 心理老师已开始记录专业处置过程 |
| `pending_retest` | 待复测 | 已安排干预后复测，等待完成或状态更新 |
| `referral` | 转介中 | 已发起外部转介；记录转介结果后仍保持此状态，除非另行安排复测或后续流转 |
| `closed` | 已闭环 | 心理老师已完成闭环确认和归档 |

“结束本次线索处理”表示本次不形成正式预警，不新增第八种主处置状态。当前 Demo 将事项标记为非活动并从预警管理活动列表移除，但保留完整历史数据供未来学生档案读取。

活动列表、状态数量、快捷筛选和高级筛选只处理 `isActive=true` 的事项。`isActive=false` 且 `disposition=ended_without_warning` 的事项不等于已闭环，也不得删除。

## 7. 反馈状态

| 代码 | 中文名称 | 判定规则 |
|---|---|---|
| `not_requested` | 未请求 | 没有反馈记录，且未发起反馈请求 |
| `pending_feedback` | 待反馈 | 已生成班主任协作任务或已请求反馈，但尚无反馈记录且未超时 |
| `feedback_received` | 已反馈 | 至少有一条反馈，且 `hasUnreadFeedback=false` |
| `feedback_overdue` | 反馈超时 | 已超过反馈期限，仍无反馈记录 |
| `new_feedback` | 有新反馈 | 至少有一条反馈，且 `hasUnreadFeedback=true` |

业务不变量：

- 有反馈不等于有新反馈。
- 只有心理老师尚未查看的反馈才显示“有新反馈”。
- 班主任每次提交反馈都新增一条记录，不覆盖历史记录。
- 多条反馈按提交时间倒序展示。
- 没有反馈记录时，不展示“查看完整反馈”入口。
- 待复核和观察中不展示班主任反馈模块。
- 请求补充反馈只能发送给班主任。

## 8. 来源类型与风险依据

### 8.1 `sourceType`

`sourceType` 表示事项最初如何被发现：

| 代码 | 中文名称 |
|---|---|
| `screening_abnormal` | 普筛异常 |
| `teacher_report` | 班主任上报 |
| `ai_chat_trigger` | AI倾诉主动触发 |

`sourceType` 可以用于来源说明和时间线，不作为风险依据标签。

### 8.2 `evidenceTypes`

`evidenceTypes` 表示心理老师复核时实际查看的证据，可以多选：

| 代码 | 中文名称 |
|---|---|
| `teacher_report` | 班主任上报 |
| `ai_chat` | AI倾诉 |
| `deep_assessment` | 深度测评 |

普筛异常不属于风险依据。风险依据模块和高级筛选中的线索类型只能使用 `evidenceTypes`。

不得继续用单一 `clueType` 同时表达来源和证据。

## 9. 系统提示等级与确认等级

- `suggestedRiskLevel`：系统、AI 或测评提示的风险等级，仅作为专业判断参考。
- `confirmedRiskLevel`：心理老师完成正式预警确认时选择的最终风险等级。
- `riskLevelAdjustmentReason`：确认等级与系统提示不同时必填的调整理由。

有效展示风险等级为：

```ts
confirmedRiskLevel ?? suggestedRiskLevel
```

只有心理老师确认正式预警后，`confirmedRiskLevel` 才有值。正式预警及其后续状态必须有 `confirmedRiskLevel`。系统不得自行把提示等级写成正式确认等级。

## 10. 系统自动动作

系统可以自动执行并留痕的动作包括：

- 将三类上游来源纳入线索池并维护线索池进度。
- 根据心理老师操作向学生生成补充评估任务。
- 学生完成必要补充评估后形成待复核预警事项。
- 心理老师确认正式预警后，向对应班主任的小程序同步协作任务，并把反馈状态更新为待反馈。
- 安排复测后记录复测时间，并在小程序生成班主任待复测提醒。
- 在复测前一天提醒班主任，由班主任提醒学生按时复测。
- 发起转介、更新复测状态和完成闭环后，向对应班主任发送小程序通知；年级主任暂不接收这些通知。
- 根据已发生的业务事件写入时间线；不得预生成未来事件。
- 根据反馈记录、是否已读和反馈期限派生反馈状态。

系统自动动作不得自行确认正式预警、确定正式风险等级、判断复测后的主状态、决定转介或完成闭环。

## 11. 必须由心理老师判断的内容

- 是否结束本次线索处理、继续观察或确认正式预警。
- 正式预警的最终风险等级。
- 系统提示等级与确认等级不同时的调整理由。
- 是否记录干预、安排复测或发起转介。
- 查看复测结果后选择风险解除并闭环、继续干预或转介。
- 如何解释和使用外部转介结果。
- 是否满足闭环归档条件。

打开详情抽屉或浏览风险依据不等于完成复核。心理老师提交三种复核结果之一时才算完成：结束本次线索处理、继续观察、确认正式预警。

## 12. 隐私边界

- 完整测评详情、完整 AI 对话和完整班主任反馈不在默认列表展示。
- 默认列表只展示学生信息、有效风险等级、当前状态、最新动态、发生时间、反馈状态和操作。
- 来源类型、证据类型和负责心理老师不在默认列表展示。
- 班主任和年级主任只获得完成各自协作职责所必需的信息。
- 校级管理者只查看脱敏、聚合后的趋势和进度。
- 当前 Demo 不实现真实权限系统，但 UI、mock 数据和文档不得突破上述边界。

## 13. 领域不变量检查

每次开发前必须确认：

1. 线索池进度没有被当作预警管理主状态。
2. 待复核事项已经完成必要补充评估并形成风险依据。
3. `sourceType` 和 `evidenceTypes` 分开建模。
4. 普筛异常没有出现在风险依据中。
5. `suggestedRiskLevel` 和 `confirmedRiskLevel` 分开建模。
6. 默认列表仍然只有七个既定字段。
7. 系统自动动作没有替代心理老师的专业判断。
8. 业务记录按时间倒序展示，且只展示已发生事件。
9. 转介结果不会自动生成复测结果；只有“安排复测”才能进入待复测。

## 14. Phase 4.6 协作、复测与归档不变量

### 14.1 班主任协作任务

- 首次班主任反馈任务在心理老师确认正式预警时生成。
- 确认时必须同时记录补充反馈要求和晚于确认时间的反馈截止时间。
- 协作对象固定为学生班主任；`responsibleTeacher` 表示负责心理老师，不得复用为班主任。
- 班主任姓名和手机号分别由 `headTeacherName`、`headTeacherPhone` 表达；当前 Demo 只展示脱敏手机号。
- 年级主任只能查看协作进度和督办，不得成为反馈请求对象。

### 14.2 有效反馈状态

- 有反馈记录时，`hasUnreadFeedback=true` 表示有新反馈，否则表示已反馈。
- 无反馈记录且无反馈任务时为未请求。
- 无反馈记录且当前时间未超过截止时间时为待反馈。
- 无反馈记录且当前时间超过截止时间时为反馈超时。
- 反馈任务进行中不得重复请求；超时后可以重新请求；已有反馈后当前版本不再提供追加一轮反馈。
- 列表、筛选、反馈模块和操作区必须使用同一套有效反馈状态规则。

### 14.3 复测和归档

- 安排复测必须记录至少一项复测量表，计划时间必须晚于系统记录的安排时间。
- 复测结果是客观记录；心理老师根据结果作出的主状态判断由“更新状态”操作承接。
- 查看复测结果、完整复测记录和归档均为只读行为，不新增业务时间线。
- 归档保留事项概况、风险依据、全部反馈请求和反馈、干预、复测、转介及完整时间线，记录不可修改或删除。
- 学生档案是归档记录的后续长期入口；Phase 5.2A 实现只读核心摘要，归档跨模块定位仍只保留回调契约。

## 15. 学生档案模块边界

- 当前事项的专业操作统一在预警管理完成；学生档案不得修改预警、干预、复测、转介或闭环状态。
- 学生档案负责按学生和事项 ID 长期聚合、查询与追溯来源业务记录，不建立可独立编辑的第二份副本。
- 干预记录是业务记录类型，展示在预警详情、归档和学生档案中，不再作为一级页面。
- 当前 MVP 中所有心理老师均可查看只读学生档案；该规则不授予跨负责人预警操作权限。
- 班主任、年级主任和校级管理者不进入个体学生档案。
- 当前风险和状态只来自未闭环且未结束的活动事项；历史闭环状态不得冒充当前状态。
- 当前班级与历史发生时班级分开保存，转班、升年级、毕业和离校不得覆盖历史记录。
- 学生身份和学籍历史属于基础数据；当前风险、当前状态、负责心理老师、干预历史和事项数量必须从共享 warning 数据派生。
- 当前 Demo 中一条 `WarningItem` 代表一条连续事项，`WarningItem.id` 即文档中的 `caseId`；不得新增含义重复的第二个事项 ID。
- 活动事项满足 `isActive=true` 且 `currentStatus!=closed`；已闭环事项与 `isActive=false` 的结束线索处理事项均属于历史。
- 当前假设每名学生最多一条活动事项；聚合发现多条时必须报告数据异常，同时可临时展示业务时间最新的一条。
- Phase 5.2A 只实现共享数据、派生摘要以及学生信息、当前概况、活动事项、学籍动态和历史事项摘要。

## 16. Phase 5.2A.1 数据语义

- 风险等级增加 `low`，表示低风险系统提示或未形成正式预警事项的事实等级。
- `confirmedRiskLevel` 仍只能由心理老师选择 `medium`、`high` 或 `critical`；正式预警确认不得选择 `low`。
- 当前风险、预警状态和当前负责人只来自当前活动事项。
- 事项来源、是否形成正式预警和是否有干预记录读取学生全部关联事项。
- 是否形成正式预警仅以存在 `confirmedRiskLevel` 为准；正式预警及后续状态缺失该字段属于数据异常。
- 本轮唯一允许的跨模块路径是学生档案查看对应预警详情并返回原档案上下文。
