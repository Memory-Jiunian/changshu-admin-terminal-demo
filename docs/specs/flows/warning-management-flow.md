# 预警管理流程规格

## 1. 适用范围

本文档定义上游线索进入路径、待复核形成条件、七种预警主状态、允许操作、系统副作用、通知对象和业务时间线规则。

线索池是上游业务容器，不属于当前预警管理 Demo 页面。预警管理从补充评估完成并形成待复核事项开始。

## 2. 三条上游来源路径

### 2.1 路径 A：普筛异常

普筛异常进入线索池
→ 心理老师发起补充评估
→ 系统向学生生成深度测评 + AI倾诉任务
→ 学生完成补充评估
→ 形成待复核预警事项
→ 心理老师提交复核结果

### 2.2 路径 B：班主任上报

班主任上报异常线索
→ 进入线索池
→ 心理老师发起补充评估
→ 系统向学生生成深度测评 + AI倾诉任务
→ 学生完成补充评估
→ 形成待复核预警事项
→ 心理老师提交复核结果

### 2.3 路径 C：AI倾诉主动触发

AI倾诉触发风险线索
→ 进入线索池
→ 心理老师发起补充评估
→ 学生完成深度测评
→ 形成待复核预警事项
→ 心理老师提交复核结果

### 2.4 补充评估与复测边界

正式预警前的深度测评和 AI倾诉属于补充评估，不称为复测。

复测只指干预后用于判断干预效果的再次测评。只有心理老师执行“安排复测”后，事项才进入待复测流程。

## 3. 心理老师复核

打开详情抽屉、浏览摘要或查看证据不等于完成复核。

心理老师提交以下任一结果时，才视为完成复核：

- 结束本次线索处理
- 继续观察
- 确认正式预警

业务时间线不记录没有结果的独立“心理老师复核”节点。复核结果直接记录为“结束本次线索处理”“标记继续观察”或“确认正式预警”。

“结束本次线索处理”表示本次不形成正式预警，不新增主处置状态。操作成功后事项转为非活动，保留完整历史数据并从预警管理活动列表、状态数量和筛选结果中移除。

## 4. 确认正式预警

待复核或观察中点击“确认正式预警”后：

1. 打开确认弹窗。
2. 展示 `suggestedRiskLevel` 系统提示等级。
3. 心理老师必须选择正式风险等级。
4. 正式等级与系统提示不同时，必须填写调整理由。
5. 正式等级相同时，判断说明可选填。
6. 确认后 `currentStatus` 变为 `formal_warning`。
7. `confirmedRiskLevel` 写入心理老师选择结果。
8. 系统自动向对应班主任生成协作任务。
9. `feedbackStatus` 变为 `pending_feedback`，`hasUnreadFeedback` 变为 `false`。
10. 最新动态和发生时间更新，列表与抽屉同步刷新。
11. 时间线只追加一条合并记录“确认正式预警”。

合并记录描述必须包含：

- 心理老师完成复核
- 正式确认的风险等级
- 风险等级调整理由（如有）
- 系统已同步生成班主任协作任务

不得再单独记录“心理老师复核”或“系统同步班主任协作任务”。

## 5. 正式预警后的流程

### 5.1 请求补充反馈

请求补充反馈只能发送给班主任。年级主任不能作为补充反馈对象。

首次确认正式预警后，系统已经自动生成班主任协作任务。“请求补充反馈”只表示心理老师后续额外请求新的事实观察，必须单独写入时间线。

### 5.2 干预

正式预警
→ 记录干预
→ 干预中
→ 新增一次或多次干预记录

干预记录按 `occurredAt` 倒序展示，最新记录在顶部。

### 5.3 待复测

安排复测
→ 系统记录复测时间
→ 小程序生成班主任待复测提醒
→ 复测前一天提醒班主任
→ 班主任提醒学生按时复测
→ 学生完成复测
→ 心理老师查看复测结果
→ 心理老师更新状态
→ 风险解除并闭环 / 继续干预 / 转介

“查看复测结果”属于阅读或审计行为，不写入学生处置业务时间线。复测记录按安排时间倒序展示。

### 5.4 转介

发起转介
→ 转介中
→ 记录转介结果

记录转介结果后：

- 主状态保持转介中。
- 不自动生成复测结果。
- 不自动进入待复测。
- 如需复测，必须另行点击“安排复测”。

Phase 5.2B.1 将“记录转介结果”扩展为“新增转介跟进”。同一转介可以追加多条跟进；每次追加更新最新动态并写入“新增转介跟进”时间线，主状态仍为转介中，不自动复测或闭环。

班主任反馈通过 `requestId` 形成协作轮次。每条实际反馈统一派生“班主任提交观察反馈”时间线，稳定 ID 为 `TL-FEEDBACK-{feedbackRecord.id}`，与已有时间线按 ID 去重并倒序展示。

转介中
→ 安排复测
→ 待复测
→ 执行完整待复测流程

### 5.5 闭环

学生完成复测
→ 心理老师读取复测结果
→ 更新复测状态为“风险解除并闭环”
→ 完成闭环归档
→ 已闭环

## 6. 通知对象

以下动作完成后，向对应班主任发送小程序通知：

- 确认正式预警并生成首次协作任务
- 发起转介
- 更新复测状态
- 完成闭环

年级主任暂不接收上述通知。当前 Demo 只在业务记录描述中体现，不接真实小程序通知。

## 8. 业务时间线规则

业务时间线按 `occurredAt` 倒序展示：最新记录在顶部，最旧记录在底部。

### 8.1 合并事件

- “普筛异常触发 + 进入线索池”合并为“普筛异常进入线索池”。描述：普筛结果达到关注阈值，系统自动纳入线索池。
- “班主任上报异常 + 进入线索池”合并为“班主任上报异常线索”。描述：班主任提交异常观察，系统已纳入线索池。
- “确认正式预警 + 系统同步班主任协作任务”合并为“确认正式预警”，描述包含心理老师确认等级和系统协作任务。

### 8.2 删除事件

业务时间线不得包含：

- 心理老师复核
- 查看复测结果
- 系统同步班主任协作任务（独立节点）

### 8.3 保留事件

- 普筛异常进入线索池
- 班主任上报异常线索
- AI倾诉触发风险线索
- 发起补充评估
- 补充评估完成
- 标记继续观察
- 结束本次线索处理
- 确认正式预警
- 请求补充反馈
- 班主任提交反馈
- 新增干预记录
- 安排复测
- 班主任待复测提醒
- 学生完成复测
- 更新复测状态
- 发起转介
- 记录转介结果
- 完成闭环归档

### 8.4 通用约束

1. 每个学生只展示已经发生的事件，不补齐未来节点。
2. 事件必须包含 `occurredAt`、操作人或系统主体和事实描述。
3. 时间线必须与 `sourceType` 和当前状态一致。
4. 补充评估完成后才能形成待复核事项。
5. 记录转介结果不能生成任何复测事件；复测事件只能从“安排复测”开始。
6. 班主任反馈、干预记录和复测记录也都按各自业务时间倒序展示。

## 9. Phase 4 操作副作用

### 9.1 活动事项

- 活动预警满足 `isActive=true`，列表、Tab 数量、快捷筛选和高级筛选只处理活动事项。
- 结束线索后设置 `isActive=false`、`disposition=ended_without_warning`，记录结束时间和原因。
- 非活动事项不删除，不映射为已闭环，未来学生档案可读取其历史数据。

### 9.2 记录与反馈

- 继续观察保存观察说明和下次复核时间，新增“标记继续观察”。
- 请求补充反馈保存要求和截止时间，固定面向班主任；状态保持正式预警，反馈状态变为待反馈，历史反馈不删除。
- 记录干预追加干预记录；正式预警首次记录后进入干预中，干预中追加记录后状态不变。
- 安排复测追加未完成复测记录并进入待复测，不预生成完成或结果事件。
- 发起转介追加未完成转介记录并进入转介中；记录结果只更新最近一条未完成转介记录，状态仍为转介中。
- 只有最近复测记录已经完成，心理老师才能更新复测状态为闭环、继续干预或转介。

### 9.3 通知与阅读行为

确认正式预警、安排复测、发起转介、更新复测状态和完成闭环的时间线描述体现“系统已同步通知对应班主任”。年级主任不接收这些通知，当前 Demo 不接真实小程序。

以下行为不写入业务时间线，也不更新最新动态：打开抽屉、切换全屏、查看复测结果、查看归档记录、查看深度测评记录、查看 AI 倾诉记录、折叠或展开模块。

## 10. Phase 4.6 最终协作规则

### 10.1 确认正式预警与首次反馈任务

待复核或观察中确认正式预警时，心理老师必须提交正式风险等级、补充反馈要求和反馈截止时间。截止时间必须晚于确认时间。成功后写入确认等级、反馈要求和截止时间，主状态变为正式预警，有效反馈状态变为待反馈，系统自动生成对应班主任协作任务。

时间线只新增一条“确认正式预警”，描述必须合并正式等级、等级调整理由（如有）、反馈要求、截止时间及系统已生成班主任协作任务。不得再单独新增协作任务节点。

### 10.2 反馈任务时效与再次请求

| 条件 | 有效反馈状态 | 底部操作 |
|---|---|---|
| 无反馈、无请求 | 未请求 | 请求补充反馈 |
| 无反馈、当前时间未超过截止时间 | 待反馈 | 禁用并显示“等待反馈” |
| 无反馈、当前时间超过截止时间 | 反馈超时 | 重新请求反馈 |
| 有反馈且已读 | 已反馈 | 不显示请求操作 |
| 有反馈且未读 | 有新反馈 | 不显示请求操作 |

重新请求时固定面向班主任，保存新的要求和截止时间，保留历史请求及反馈记录，主状态保持正式预警，并新增“重新请求反馈”时间线。反馈模块只展示状态、截止时间、协作对象和反馈内容，不提供重复提醒或重发入口。

### 10.3 安排复测

安排时间由系统自动记录且不可编辑。心理老师必须选择至少一项复测量表，并填写晚于安排时间的计划复测时间；补充说明可选。成功后复测记录和“安排复测”时间线描述均保存量表名称。

复测结果弹窗只展示计划时间、完成时间、量表、结果摘要和与上次结果对比。心理老师结论不在结果弹窗中展示，后续专业状态判断由“更新状态”操作承接。查看完整复测记录和归档均不写业务时间线。

### 10.4 完整归档

已闭环事项的归档为只读完整记录，包括事项概况、学生信息、风险等级及调整理由、风险依据、全部反馈请求与反馈、全部干预记录、全部复测记录、全部转介记录和倒序完整时间线。查看归档不修改状态、记录或时间线。

### 10.5 工作台定向入口

工作台可以按 `warningId` 打开预警详情并定位概况、风险依据、反馈、复测、转介或操作区。定位不改变业务状态，也不自动标记反馈已读。心理老师必须在反馈模块明确点击“标记为已查看”；工作台和普通预警入口调用同一共享 action，更新反馈记录阅读时间和兼容字段，不写业务时间线。
