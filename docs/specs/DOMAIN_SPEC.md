# 预警管理领域规格

## 1. 文档目的

本文档定义预警管理 Demo 的稳定领域语言、角色边界和业务不变量。页面行为以 `PRD.md` 为依据，状态流转以 `docs/specs/flows/warning-management-flow.md` 为依据；发生冲突时按 `AGENTS.md` 中的规格优先级处理。

## 2. 项目目标

管理终端是校园心理风险处置工作台，核心链路为：发现风险、专业复核、确认正式预警、记录干预、安排复测或转介、完成闭环归档。

系统只负责汇集事实、呈现状态、记录过程和提供操作入口，不替代心理老师做专业心理判断。

当前作品集 Demo 只覆盖预警管理默认列表和学生风险详情抽屉，使用 mock 数据，不接真实后端、登录、权限系统或小程序通知服务。

## 3. 三端分工

| 终端 | 主要职责 | 不承担的职责 |
|---|---|---|
| 学生端 | 普筛、深度测评、AI 倾诉，产生测评结果和风险线索 | 不展示预警处置流程，不进行专业风险定级 |
| 小程序端 | 向班主任、年级主任触达协作任务和提醒；收集事实观察、反馈进度和督办留痕 | 不确认正式预警，不修改风险等级或主处置状态，不展示完整测评和完整 AI 对话 |
| 管理终端 | 汇集线索、测评摘要和协作反馈；由心理老师完成复核、干预、复测、转介和闭环管理 | 不作为学生测评入口，不替代小程序做移动协作，不替代专业判断 |

## 4. 角色与权限边界

| 角色 | 可以查看 | 可以操作 | 明确禁止 |
|---|---|---|---|
| 心理老师 | 职责范围内的学生信息、测评摘要、AI 线索摘要、班主任反馈、时间线 | 复核、驳回、继续观察、确认正式预警、请求补充反馈、记录干预、安排复测、发起与记录转介、闭环归档 | 不得把系统输出当作自动专业结论 |
| 班主任 | 小程序中的协作任务和完成任务所需的有限学生信息 | 上报异常、提交事实观察和反馈、提醒学生复测 | 不得判断风险等级、确认正式预警、修改主处置状态、查看完整测评或完整 AI 对话 |
| 年级主任 | 小程序中的协作或督办事项及必要的有限信息 | 提交协作反馈或督办留痕 | 不得判断风险等级、确认正式预警、修改主处置状态、查看完整测评或完整 AI 对话 |
| 校级管理者 | 脱敏后的学校整体趋势、处置进度和资源调度信息 | 查看聚合态势、协调校内资源 | 不直接进入学生个人心理档案，不查看完整学生风险详情 |

班主任和年级主任不是当前管理终端的主用户。

## 5. 主处置状态

主处置状态与反馈状态是两个独立维度，不得互相替代。

| 代码 | 中文名称 | 含义 |
|---|---|---|
| `pending_review` | 待复核 | 线索已进入待复核线索池，等待心理老师判断 |
| `observing` | 观察中 | 暂不形成正式预警，心理老师继续观察 |
| `formal_warning` | 正式预警 | 心理老师已确认形成正式预警 |
| `in_intervention` | 干预中 | 心理老师已开始记录专业处置过程 |
| `pending_retest` | 待复测 | 已安排复测，等待复测完成、查看结果或更新状态 |
| `referral` | 转介中 | 已发起外部转介，等待或记录外部转介结果 |
| `closed` | 已闭环 | 心理老师已完成闭环确认和归档 |

“普筛异常”是流程触发事件，“协作反馈中”是协作进度描述，二者都不是主处置状态。

“驳回”后的持久化状态尚未在七种主状态中定义，见流程规格的“待确认问题”。

## 6. 反馈状态

| 代码 | 中文名称 | 判定规则 |
|---|---|---|
| `not_requested` | 未请求 | 没有反馈记录，且未发起反馈请求 |
| `pending_feedback` | 待反馈 | 已请求反馈，但尚无反馈记录且未超时 |
| `feedback_received` | 已反馈 | 至少有一条反馈，且 `hasUnreadFeedback=false` |
| `feedback_overdue` | 反馈超时 | 已超过反馈期限，仍无反馈记录 |
| `new_feedback` | 有新反馈 | 至少有一条反馈，且 `hasUnreadFeedback=true` |

业务不变量：

- 有反馈不等于有新反馈。
- 只有心理老师尚未查看的反馈才显示“有新反馈”。
- 班主任每次提交反馈都新增一条记录，不覆盖历史记录。
- 多条反馈按提交时间倒序展示。
- 没有反馈记录时，不展示“查看完整反馈”入口。
- 待复核和观察中不展示班主任反馈模块。

## 7. 风险依据线索类型

详情抽屉“风险依据”中的线索类型只允许：

| 代码 | 中文名称 |
|---|---|
| `teacher_report` | 班主任上报 |
| `ai_chat` | AI倾诉 |
| `deep_assessment` | 深度测评 |

`screening_abnormal` / 普筛异常只表示流程触发或时间线事件，不是风险依据线索类型。即使旧数据、内部类型或高级筛选仍保留该代码，详情风险依据也不得展示“普筛异常”。

线索类型不得出现在默认列表，只能出现在详情抽屉和高级筛选中。

## 8. 系统自动动作

系统可以自动执行并留痕的动作包括：

- 普筛达到关注阈值后，记录“普筛异常触发”。
- 将普筛异常或班主任上报异常纳入待复核线索池。
- 在心理老师确认正式预警后，向对应班主任的小程序同步协作任务。
- 记录复测时间，并在小程序生成班主任待复测提醒。
- 在复测前一天提醒班主任，由班主任提醒学生按时复测。
- 根据已发生的业务事件写入时间线；不得预生成未来事件。
- 根据反馈记录、是否已读和反馈期限派生反馈状态。

系统自动动作不得自行确认正式预警、判断复测后的主状态、决定转介或完成闭环。

## 9. 必须由心理老师判断的内容

- 风险依据是否足以形成正式预警。
- 驳回线索、继续观察或确认正式预警。
- 风险等级及其后续专业处置。
- 是否记录干预、安排复测或发起转介。
- 查看复测结果后选择风险解除并闭环、继续干预或转介。
- 如何解释和使用外部转介结果。
- 是否满足闭环归档条件。

默认列表只能呈现事实，不得展示“下一步动作”或系统生成的专业处置建议。

## 10. 隐私边界

- 完整测评详情、完整 AI 对话和完整班主任反馈不在默认列表展示。
- 默认列表只展示学生信息、风险等级、当前状态、最新动态、发生时间、反馈状态和操作。
- 负责心理老师不在默认列表展示，只能用于详情和高级筛选。
- 班主任和年级主任只获得完成协作任务所必需的信息。
- 校级管理者只查看脱敏、聚合后的趋势和进度，不查看完整个体风险详情。
- 当前 Demo 不实现真实权限系统，但 UI、mock 数据和文档不得突破上述边界。

## 11. 领域不变量检查

每次开发前必须确认：

1. 主处置状态和反馈状态分开建模。
2. 普筛异常没有被当作详情风险依据线索类型。
3. 默认列表仍然只有七个既定字段。
4. 系统自动动作没有替代心理老师的专业判断。
5. 时间线只展示已发生事件，且与当前状态和来源路径一致。
6. 转介结果不会自动生成复测结果；只有“安排复测”才能进入待复测。
