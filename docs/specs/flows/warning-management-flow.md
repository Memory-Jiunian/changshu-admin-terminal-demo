# 预警管理流程规格

## 1. 适用范围

本文档定义预警管理详情抽屉七种主状态的来源路径、允许操作、状态变化、通知和时间线规则。当前阶段只建立规格，不实现按钮后的本地状态变化。

## 2. 线索进入路径

### 2.1 普筛路径

普筛异常触发
→ 普筛异常进入待复核线索池
→ 心理老师发起补充评估
→ 深度测评 + AI倾诉完成
→ 心理老师复核

规则：

- “普筛异常触发”必须带触发时间。
- 普筛异常是流程事件，不是风险依据线索类型。
- 学生尚未完成后续节点时，时间线只展示已经发生的前置节点。

### 2.2 班主任上报路径

班主任上报异常
→ 异常线索进入待复核线索池
→ 心理老师发起补充评估
→ 深度测评 + AI倾诉完成
→ 心理老师复核

规则：

- 该路径起点必须是“班主任上报异常”，不得改称“普筛异常”。
- “发起补充评估”必须位于“深度测评 + AI倾诉完成”之前。

## 3. 复核后的主流程

心理老师复核后可选择：

- 驳回：不形成正式预警并归档；归档在七种主状态中的具体表示待确认。
- 继续观察：进入或保持“观察中”。
- 确认正式预警：进入“正式预警”，系统同步生成班主任协作任务。

确认正式预警后的默认班主任协作任务是系统自动动作。“请求补充反馈”只表示心理老师后续额外请求事实观察，不能与默认协作任务混为一谈。

后续主流程：

正式预警
→ 记录干预
→ 干预中
→ 安排复测或发起转介

待复测路径：

安排复测
→ 系统记录复测时间
→ 小程序生成班主任待复测提醒
→ 复测前一天提醒班主任
→ 班主任提醒学生按时复测
→ 学生完成复测
→ 心理老师查看复测结果
→ 心理老师更新状态
→ 风险解除并闭环 / 继续干预 / 转介

转介路径：

发起转介
→ 转介中
→ 记录转介结果

转介结果只记录外部反馈，不自动生成复测结果。需要复测时必须另行点击“安排复测”：

转介中
→ 安排复测
→ 待复测
→ 执行完整待复测流程

闭环路径：

查看复测结果
→ 更新复测状态为“风险解除并闭环”
→ 完成闭环归档
→ 已闭环

## 5. 时间线规则

允许的核心事件池：

- 普筛异常触发
- 普筛异常进入线索池
- 班主任上报异常
- 异常线索进入待复核线索池
- 发起补充评估
- 深度测评 + AI倾诉完成
- 心理老师复核
- 标记继续观察
- 确认正式预警
- 系统同步班主任协作任务
- 请求补充反馈
- 班主任提交反馈
- 新增干预记录
- 安排复测
- 小程序生成班主任待复测提醒
- 复测前一天提醒班主任
- 学生完成复测
- 查看复测结果
- 更新复测状态
- 发起转介
- 记录转介结果
- 完成闭环归档

通用约束：

1. 每个学生只展示已经发生的事件，不补齐未来节点。
2. 事件必须包含时间、操作人或系统主体，并按时间顺序展示。
3. 时间线必须与来源路径和当前状态一致。
4. 普筛路径必须保留“普筛异常触发”的时间。
5. 班主任上报路径不得出现伪造的普筛触发节点。
6. “深度测评 + AI倾诉完成”之前必须已有“发起补充评估”。
7. 确认正式预警后系统自动生成协作任务；是否使用独立时间线节点见待确认问题。
8. 记录转介结果不能生成任何复测事件；复测事件只能从“安排复测”开始。

## 6. 待确认问题

以下问题在现有文档或实现中没有唯一答案，进入 Phase 4 前必须确认：

1. 驳回后应使用哪一个持久化主状态。现有七状态没有“已驳回 / 已归档 / 低风险”。
2. 转介中记录转介结果后，主状态应保持转介中、进入干预中，还是允许直接闭环。已确认的是不得自动进入待复测。
3. “系统同步班主任协作任务”应作为独立时间线节点，还是合并到“确认正式预警”的描述中。
4. “请求补充反馈”的协作对象是否只能是班主任，还是允许选择年级主任。
5. 是否允许 AI倾诉直接进入待复核线索池。如果允许，需要补充第三条来源路径和对应入池事件；当前 mock 中存在此类历史样例。
6. 发起转介、更新复测状态和完成闭环后，是否需要向班主任或年级主任发送小程序通知。
