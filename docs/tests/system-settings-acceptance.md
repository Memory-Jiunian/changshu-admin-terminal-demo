# 系统设置 MVP 验收基线

## 学校与组织

- [ ] 学校名称修改后 Topbar 同步。
- [ ] 学期修改后校级总览默认周期同步。
- [ ] 学期开始晚于结束时阻止保存。
- [ ] 年级名称校内唯一，班级名称在同年级唯一。
- [ ] 班主任只能选择启用教师。
- [ ] 被班级或学生引用的组织不提供硬删除。

## 教师与学生

- [ ] 工号和学号分别唯一。
- [ ] 教师支持多角色。
- [ ] 活动预警负责人、当前班主任和未来预约负责人不能停用。
- [ ] 转班关闭旧学籍区间并新增历史。
- [ ] 毕业和离校从默认在校列表移除，但历史事项保留。
- [ ] 当前资料更新不覆盖 Warning、反馈、预约、干预、复测和转介快照。

## 导入

- [ ] `.xlsx` 与 `.csv` 固定模板可以解析。
- [ ] 预览前不修改共享数据。
- [ ] Error 阻止确认；Warning 可主动确认；相同内容为 Skip。
- [ ] insert_only 遇到已有自然键为 Error。
- [ ] upsert 保留内部 ID 并展示字段差异。
- [ ] 预览后版本变化要求重新预览。
- [ ] 任一应用失败整批回滚。
- [ ] 重复确认不重复导入。
- [ ] 文件缺少现有记录不自动删除或离校。

## 跨模块与审计

- [ ] 学生当前资料同步档案和总览。
- [ ] 教师与班主任变更只影响当前视图和新业务默认值。
- [ ] 系统设置操作只写 `AdminAuditLog`。
- [ ] 浏览、搜索、筛选、预览和取消不写记录。
- [ ] 预警业务时间线长度和内容不因系统设置操作变化。

## 页面与回归

- [ ] 五个内部 Tab、分页、空状态、错误状态、引用冲突和离开保护可用。
- [ ] Dialog 不贴顶贴底，无整页横向滚动。
- [ ] 预警、学生档案、工作台和校级总览回归通过。
- [ ] 1440×900、1536×864、1280×720 走查通过。
- [ ] `npm test`、`npm run build`、HTTP 200、工作区干净并推送。
