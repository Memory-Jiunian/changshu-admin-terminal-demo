# 功能切片：学生心理档案查询与详情（优化版）

## 1. 切片名称
学生心理档案查询与详情

## 2. 本轮目标
心理老师能够快速定位目标学生，并在一个连续的档案视图中查看其基础信息、当前心理状态及历史测评、预警、干预、复测、转介和闭环记录，为当前专业判断提供历史依据。

## 3. 主要使用者
- 心理老师
- 当前阶段不区分“负责心理老师”和“非负责心理老师”的查看权限
- 所有心理老师均可查看学生完整心理档案
- 班主任、年级主任和校级管理者不使用该管理终端模块

## 4. 核心场景
1. 已知学生姓名或学号，快速查找档案
2. 只知道年级、班级，通过筛选定位学生
3. 从预警管理或干预记录直接进入学生档案
4. 第二次或第三次干预前查看历史处置
5. 查看学生转班、升年级、毕业等学籍动态
6. 历史记录较多时切换全屏模式集中查看

## 5. 本轮范围

### 包含
- 学生档案列表
- 姓名 / 学号搜索
- 年级 / 班级筛选
- 高级筛选
- 学生档案详情抽屉
- 全屏查看模式
- 当前心理状态概况
- 当前活动事项
- 历史测评、预警、干预、复测、转介和闭环记录
- 学籍动态
- 全量业务时间线
- 从工作台、预警管理、干预记录进入档案
- 返回来源页面时保留原有搜索、筛选、分页和抽屉上下文

### 不包含
- 学校、年级、班级组织结构维护
- 学生、班主任、年级主任账号增删改
- 批量导入导出
- 真实校务系统同步
- 真实权限审批
- 在档案页新增、编辑或删除历史业务记录
- 在档案页直接改变预警、干预、复测或转介状态
- 家长联系方式、家庭住址等非必要敏感信息

## 6. 已确认的产品决定
1. 学生档案详情默认使用右侧抽屉。
2. 抽屉宽度与预警管理详情抽屉统一。
3. 抽屉宽度以后允许调整，必须通过统一宽度变量或公共容器一次性修改全部详情抽屉。
4. 学生档案支持全屏查看；全屏不是简单放大抽屉，而是复用同一份数据并切换为更适合长内容阅读的布局。
5. 当前阶段所有心理老师均可查看全部学生档案，不实现“无权限查看”和“申请查看”。
6. 学生档案列表不展示负责心理老师。
7. 列表只展示学生基础定位信息，风险、状态和历史记录在详情中查看。
8. 高级筛选可以使用未展示为列表列的数据，例如风险状态、是否有历史干预和负责心理老师。
9. 学生转班、升年级和毕业不能覆盖历史信息，必须通过“学籍动态”保存发生时间与历史班级。
10. 历史记录统一按最新在前展示。
11. 测评、预警、干预、复测、转介和闭环记录必须按同一业务事项建立关联，避免只看到孤立记录。

## 7. 上下游

### 上游入口
- 左侧导航
- 工作台
- 预警管理
- 干预记录

### 下游影响
- 为预警复核提供历史依据
- 为后续干预计划提供历史依据
- 为复测结果提供前后对比
- 为转介和闭环判断提供过程证据
- 为作品集展示跨模块信息聚合能力提供完整案例

## 8. 完成标准
1. 能按姓名或学号查找学生。
2. 能按年级、班级定位学生。
3. 能通过高级筛选查找有活动预警、有历史干预或特定负责心理老师的学生。
4. 同名学生可通过学号、年级和班级区分。
5. 列表只展示基础信息，不暴露完整心理记录。
6. 能从其他业务模块直接打开正确学生档案。
7. 返回来源页面时保留原有上下文。
8. 能区分当前活动事项与历史事项。
9. 能查看测评、预警、干预、复测、转介和闭环记录。
10. 能看出同一事项中“干预—复测—继续干预 / 转介 / 闭环”的关系。
11. 能查看学生学籍动态。
12. 标准抽屉和全屏模式数据一致，布局分别适配对应阅读场景。
13. 档案页不直接新增、修改或删除来源业务记录。

## 9. 暂不阻塞 MVP 的问题
1. 是否需要批量导出学生档案。
2. 是否需要按测评类型进一步筛选。
3. 是否需要在全屏模式提供打印视图。
4. 是否需要在后续真实项目中恢复负责制和授权查看。
